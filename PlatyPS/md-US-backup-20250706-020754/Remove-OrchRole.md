# Remove-OrchRole

## SYNOPSIS
Removes roles from the Orchestrator.

## SYNTAX

```
Remove-OrchRole -Name <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRIPTION
The Remove-OrchRole cmdlet permanently removes roles from the Orchestrator environment. Roles define permissions and access levels for users and groups within the organization.

This cmdlet can remove both custom roles and system roles, though care should be taken when removing system roles as they may be required for proper functioning of the Orchestrator environment. The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before deletion.

Only roles that are not currently assigned to users or groups can be removed. If a role is in use, you must first remove all assignments before the role can be deleted.

Primary Endpoint: DELETE /odata/Roles({roleId})

OAuth required scopes: OR.Users

Required permissions: Roles.Delete

## EXAMPLES

### Example 1
```powershell
PS C:\> Remove-OrchRole CustomRole1
```

Removes the role named "CustomRole1" from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Remove-OrchRole CustomRole1, CustomRole2 -WhatIf
```

Shows what would happen if the specified roles were removed, without actually performing the removal.

### Example 3
```powershell
PS C:\> Remove-OrchRole Test* -Confirm
```

Removes all roles whose names start with "Test", prompting for confirmation before each removal.

### Example 4
```powershell
PS C:\> Remove-OrchRole TemporaryRole -Path Orch1:, Orch2:
```

Removes the role named "TemporaryRole" from multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchRole | Where-Object {$_.Type -eq "Custom" -and $_.IsEditable -eq $true} | Remove-OrchRole -WhatIf
```

Identifies custom, editable roles and shows what would be removed without actually performing the deletion.

## NOTES

This cmdlet operates on tenant entities, which means it affects roles across the entire Orchestrator tenant rather than specific folders.

Before removing roles, ensure they are not assigned to any users or groups. You can use Get-OrchRole to check role assignments and Remove-OrchRoleFromUser or Remove-OrchRoleFromFolderUser to remove assignments if necessary.

System roles that are marked as IsStatic = $true typically should not be removed as they are essential for Orchestrator functionality.

Use the -WhatIf parameter to preview the operation before actual execution, especially when using wildcard patterns that might match multiple roles.

The removal operation is irreversible. Once a role is deleted, it cannot be recovered and must be recreated if needed.
