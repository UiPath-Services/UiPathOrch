# Get-OrchLibrary

## SYNOPSIS
Gets library packages from Orchestrator or external host feeds.

## SYNTAX

```
Get-OrchLibrary [[-Id] <String[]>] [-HostFeed] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchLibrary cmdlet retrieves library packages from the Orchestrator environment. Libraries are reusable components that contain activities, workflows, and other automation resources that can be shared across multiple processes.

By default, this cmdlet returns libraries from the connected Orchestrator instance. When the -HostFeed parameter is specified, it retrieves packages from external host feeds (such as the official NuGet gallery) that can be imported into Orchestrator.

The cmdlet returns essential information about each library including its ID, version, description, publication date, and metadata such as authors, tags, and project type.

Primary Endpoint: GET /odata/Libraries

OAuth required scopes: OR.Execution or OR.Execution.Read

Required permissions: Libraries.View

## EXAMPLES

### Example 1
```powershell
PS C:\> Get-OrchLibrary
```

Retrieves all library packages from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Get-OrchLibrary UiPath.Excel.Activities
```

Retrieves the specific library with the ID "UiPath.Excel.Activities".

### Example 3
```powershell
PS C:\> Get-OrchLibrary -HostFeed *Excel*
```

Searches for Excel-related libraries from external host feeds using wildcard matching.

### Example 4
```powershell
PS C:\> Get-OrchLibrary -Path Orch1:, Orch2:
```

Retrieves libraries from multiple Orchestrator environments (Orch1 and Orch2).

### Example 5
```powershell
PS C:\> Get-OrchLibrary | Select-Object Id, Version, Description | ConvertTo-Json -Depth 3
```

Retrieves all libraries and displays their structure in JSON format for detailed analysis.

### Example 6
```powershell
PS C:\> Get-OrchLibrary *Library* | Export-Csv libraries.csv
```

Exports all libraries containing "Library" in their name to a CSV file for inventory management.

## NOTES

This cmdlet operates on tenant entities, which means it retrieves information from the entire Orchestrator tenant rather than specific folders.

When using the -HostFeed parameter, be aware that this retrieves packages from external sources and may take longer to complete depending on network conditions and the size of the external feed.

Use ConvertTo-Json with appropriate depth settings to explore the complete structure of library objects, as they contain nested properties like ResourceTags that may not be visible in the default display format.

For inventory management and reporting purposes, consider using Select-Object to choose specific properties and Export-Csv to create reports.
