# Get-OrchMachine

## SYNOPSIS
Gets machines from UiPath Orchestrator.

## SYNTAX

```
Get-OrchMachine [[-Name] <String[]>] [-Path <String[]>] [-ExpandRobotUser] [-ExportCsv <String>]
 [-CsvEncoding <Encoding>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchMachine` cmdlet retrieves machine information from UiPath Orchestrator. Machines represent the compute resources where robots execute automation processes. This includes both physical machines and machine templates used for robot provisioning.

This cmdlet returns comprehensive machine information including machine types (Standard, Template), scopes (Default, PersonalWorkspace, Serverless, etc.), license slot allocations, robot user assignments, and configuration details.

The cmdlet supports filtering by machine name, expanding robot user details, and exporting results to CSV files for reporting and analysis purposes.

Multiple values for the -Name and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: /odata/Machines&$expand=UpdateInfo

OAuth required scopes: OR.Machines or OR.Machines.Read

Required permissions: Machines.View

## EXAMPLES

### Example 1
```powershell
PS C:\> Set-Location Orch1:\
PS Orch1:\> Get-OrchMachine
```

Gets all machines in the current Orchestrator instance.

### Example 2
```powershell
PS Orch1:\> Get-OrchMachine "*workspace*"
```

Gets all machines whose names contain "workspace" using wildcard pattern matching.

### Example 3
```powershell
PS Orch1:\> Get-OrchMachine -ExpandRobotUser
```

Gets all machines with expanded robot user information and displays license slot details (UA-Slots, NP-Slots, TA-Slots).

### Example 4
```powershell
PS Orch1:\> Get-OrchMachine | Where-Object Type -eq "Standard"
```

Gets only Standard machines (physical machines) as opposed to Template machines.

### Example 5
```powershell
PS Orch1:\> $machine = Get-OrchMachine | Select-Object -First 1
PS Orch1:\> $machine | ConvertTo-Json -Depth 3
```

Gets the first machine and displays its complete structure in JSON format to explore all properties.

### Example 6
```powershell
PS Orch1:\> Get-OrchMachine -ExportCsv "C:\Reports\machines.csv"
```

Gets all machines and exports the data to a CSV file for reporting purposes.

## NOTES
- Machine types include Standard (physical machines) and Template (machine templates for robot provisioning)
- Scopes define the machine's purpose: Default, PersonalWorkspace, Serverless, AgentService, or AutomationCloud
- The -ExpandRobotUser parameter changes the display format to show license slot details
- License slots indicate capacity: UnattendedSlots (UA-Slots), NonProductionSlots (NP-Slots), TestAutomationSlots (TA-Slots)
- Personal workspace machines are automatically created for individual users
- Machine templates are used for robot autoprovisioning and serverless execution
- Use ConvertTo-Json to explore the complete object structure including nested properties
