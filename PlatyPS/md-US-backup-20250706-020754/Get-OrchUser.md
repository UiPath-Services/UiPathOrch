# Get-OrchUser

## SYNOPSIS
Gets users from UiPath Orchestrator.

## SYNTAX

```
Get-OrchUser [[-UserName] <String[]>] [[-FullName] <String[]>] [-Type <String[]>] [-ExpandDetails]
 [-Path <String[]>] [-ExportCsv <String>] [-CsvEncoding <Encoding>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchUser` cmdlet retrieves user information from UiPath Orchestrator. This includes individual users, groups, and robot accounts that can authenticate and interact with the Orchestrator environment.

By default, this cmdlet uses the list API endpoint which provides basic user information efficiently. When the -ExpandDetails parameter is specified, the cmdlet calls individual user detail APIs to retrieve comprehensive information including detailed notification preferences, session permissions, and robot provisioning settings.

For performance reasons, detailed information is not retrieved by default. However, once retrieved with -ExpandDetails, the detailed information is cached and will be available in subsequent calls to the same cmdlet even without the -ExpandDetails parameter.

The cmdlet supports filtering by username, full name, and user type, and can export results to CSV files for reporting and analysis purposes.

Multiple values for the -UserName, -FullName, and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: GET /odata/Users?$expand=OrganizationUnits,UserRoles

OAuth required scopes: OR.Users or OR.Users.Read

Required permissions: Users.View

## EXAMPLES

### Example 1
```powershell
PS C:\> Set-Location Orch1:\
PS Orch1:\> Get-OrchUser
```

Gets basic information for all users in the current Orchestrator instance.

### Example 2
```powershell
PS Orch1:\> Get-OrchUser *user*
```

Gets users whose username contains "user" using wildcard pattern matching.

### Example 3
```powershell
PS Orch1:\> Get-OrchUser -Type DirectoryUser
```

Gets only individual user accounts (not groups or robot accounts).

### Example 4
```powershell
PS Orch1:\> Get-OrchUser user@example.com -ExpandDetails
```

Gets detailed information for a specific user, including comprehensive notification preferences, robot provisioning details, and session permissions.

### Example 5
```powershell
PS Orch1:\> $user = Get-OrchUser user@example.com -ExpandDetails
PS Orch1:\> $user.NotificationSubscription
```

Gets expanded user details and displays the notification subscription preferences.

### Example 6
```powershell
PS Orch1:\> Get-OrchUser -Type "DirectoryGroup" | Select-Object UserName, FullName, RolesList
```

Gets all directory groups and displays their names and assigned roles.

## NOTES
- By default, this cmdlet uses the list API for efficient retrieval of basic user information
- The -ExpandDetails parameter calls individual user detail APIs to retrieve comprehensive information
- Detailed information retrieved with -ExpandDetails is cached for subsequent calls
- User types include DirectoryUser (individual users), DirectoryGroup (groups), and DirectoryRobot (robot accounts)
- The NotificationSubscription property (available with -ExpandDetails) contains detailed notification preferences for various events
- RobotProvision and UnattendedRobot properties contain robot configuration details when applicable
- Use ConvertTo-Json to explore the complete object structure, especially when using -ExpandDetails
- The caching mechanism improves performance by storing detailed information once retrieved
- Clear-OrchCache can be used to clear cached information if needed
