# Enable-OrchWebhook

## SYNOPSIS
Enables the webhooks.

## SYNTAX

```
Enable-OrchWebhook [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Enable-OrchWebhook cmdlet enables previously disabled webhooks in the Orchestrator environment. Webhooks allow Orchestrator to send HTTP notifications to external systems when specific events occur, enabling real-time integration with external applications and services.

When a webhook is enabled, it will start sending notifications for the events it is configured to monitor. This includes events such as job completion, process deployment, robot status changes, and other Orchestrator activities based on the webhook's configuration.

The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before enabling webhooks.

Primary Endpoint: PATCH /odata/Webhooks({webhookId})

OAuth required scopes: OR.Webhooks

Required permissions: Webhooks.Edit

## EXAMPLES

### Example 1
```powershell
PS C:\> Enable-OrchWebhook JobCompletionHook
```

Enables the webhook named "JobCompletionHook".

### Example 2
```powershell
PS C:\> Enable-OrchWebhook IntegrationHook1, IntegrationHook2
```

Enables multiple webhooks by specifying their names in a comma-separated list.

### Example 3
```powershell
PS C:\> Enable-OrchWebhook Test* -WhatIf
```

Shows what would happen if all webhooks whose names start with "Test" were enabled, without actually performing the operation.

### Example 4
```powershell
PS C:\> Enable-OrchWebhook ProductionHook -Path Orch1:, Orch2:
```

Enables the webhook named "ProductionHook" in multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchWebhook | Where-Object Enabled -eq $false | Enable-OrchWebhook -Confirm
```

Finds all disabled webhooks and enables them with confirmation prompts for each operation.

### Example 6
```powershell
PS C:\> Enable-OrchWebhook CriticalEventsHook | Select-Object Name, Enabled, Url
```

Enables a webhook and displays its updated status including name, enabled state, and URL.

## NOTES

This cmdlet operates on tenant entities, which means it affects webhooks across the entire Orchestrator tenant rather than specific folders.

Once enabled, webhooks will immediately start sending notifications based on their configured event types and filters. Ensure that the target URLs are accessible and ready to receive webhook notifications before enabling them.

Use Get-OrchWebhook to verify the current status of webhooks before and after enabling them.

Consider using -WhatIf to preview the operation, especially when using wildcard patterns that might match multiple webhooks.

If a webhook's target URL is no longer valid or accessible, enabling it may result in failed notification attempts and potential performance impacts.
