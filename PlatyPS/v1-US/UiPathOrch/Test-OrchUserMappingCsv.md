# Test-OrchUserMappingCsv

## SYNOPSIS

Validates a user mapping CSV file for cross-organization tenant migration.

## SYNTAX

### __AllParameterSets

```
Test-OrchUserMappingCsv [-CsvFile] <string> [-SourceTenant] <string> [-DestinationTenant] <string>
 [<CommonParameters>]
```

## ALIASES

## DESCRIPTION

Validates a user mapping CSV file generated by New-OrchUserMappingCsv to ensure that all source users exist in the source tenant and all destination users exist in the destination directory.

The validation performs the following checks for each entry in the CSV:

1. Verifies that the SourceUserName exists in the source tenant as either a tenant user or a folder user.
2. Checks that the DestinationUserName is not empty.
3. Searches the destination directory to confirm that the DestinationUserName exists. If the user is not found in the directory but Name/DisplayName columns are provided, it reports a warning indicating that a local user can be created using New-PmUser.

After validation, the cmdlet outputs a summary showing the count of OK, Warning, and Error results.

The -SourceTenant and -DestinationTenant parameters support tab completion. Press [Ctrl+Space] or [Tab] to see available drives.

Primary Endpoint: GET /odata/Users, GET /api/Directory/Search/{partitionGlobalId} (Identity Server)

OAuth required scopes: OR.Users or OR.Users.Read

Required permissions: Users.View

## EXAMPLES

### Example 1: Validate a user mapping CSV

```powershell
PS C:\> Test-OrchUserMappingCsv UserMapping.csv Orch1: Orch2:
```

Validates the UserMapping.csv file by checking source users against the Orch1: tenant and destination users against the Orch2: directory.

### Example 2: Validate with explicit parameter names

```powershell
PS C:\> Test-OrchUserMappingCsv -CsvFile C:\Migration\UserMapping.csv -SourceTenant Orch1: -DestinationTenant Orch2:
```

Validates the specified CSV file with explicit parameter names.

## OUTPUTS

## NOTES

This cmdlet is designed to work with CSV files generated by New-OrchUserMappingCsv. The CSV file must contain at least SourceUserName and DestinationUserName columns. Optional columns include Name, SurName, and DisplayName.
