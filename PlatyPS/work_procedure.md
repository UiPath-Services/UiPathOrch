## 🎯 品質基準

### 必須要件
1. ✅ すべての例が Orch1: ドライブで実際に実行可能
2. ✅ -Path/-Recurse パラメータは他のパラメータより前に記載
3. ✅ 例の出力が適切で意味がある
4. ✅ 例の説明が実際の動作と一致している
5. ✅ **少ない例で多くの使い方を網羅的に説明**
   - 冗長な例は削除
   - 各例は異なる使用パターンを示す
   - 複雑な使い方は不要（シンプルで実用的な例を優先）

### 例の設計原則
- **最小限の数**: 3-5例が理想的（cmdletの複雑さに応じて調整）
- **最大限の網羅性**: 各例が異なる機能・パラメータを示す
- **段階的な学習**: 基本→応用の順で配置
- **実用性重視**: 実際のシナリオで使える例

### 良い例のパターン
```powershell
# Example 1: 基本的な使い方（パラメータなし、または最小限）
Get-OrchJob

# Example 2: 単一の重要パラメータ
Get-OrchJob -State Successful

# Example 3: 複数パラメータの組み合わせ
Get-OrchJob -Path "Orch1:\Shared" -Recurse -State Successful -First 10

# Example 4: パイプライン使用
Get-OrchJob -Last Week | Where-Object Duration -gt "00:30:00"
```

### 削除すべき例のパターン
- 同じ機能を示す重複した例
- 過度に複雑な例
- 特殊すぎて一般的でない例
- 説明なしで複数のパラメータを使う例

### 検証項目
- [ ] コマンド構文が正しい
- [ ] パラメータの順序が正しい（-Path/-Recurse が最初）
- [ ] 実際に実行して動作確認
- [ ] エラーが発生しない（または適切にエラーハンドリングされている）
- [ ] 例の説明と実際の動作が一致

## 📐 パラメータ順序ルール

**重要**: -Path/-Recurse/-Depth パラメータを使用する場合、必ず他のパラメータより前に配置する

```powershell
# ✅ 正しい
Get-OrchJob -Path "Orch1:\Shared" -Recurse -State Successful

# ❌ 間違い
Get-OrchJob -State Successful -Path "Orch1:\Shared" -Recurse
```
