# Get-TmTestSet

## SYNOPSIS
Gets test sets from projects in Test Manager.

## SYNTAX

```
Get-TmTestSet [[-Name] <String[]>] [-Path <String[]>] [-Recurse] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The Get-TmTestSet cmdlet retrieves test sets from Test Manager projects. Test sets are collections of test cases that can be executed together as a group, providing organized test execution and reporting capabilities.

This cmdlet operates on folder entities and requires navigation to specific Test Manager projects or the use of -Path, -Recurse, or -Depth parameters. It works with the UiPathOrchTm provider, which is automatically available when Test Manager scopes are included in the configuration.

Test sets can originate from various sources including Orchestrator processes, external test management systems, or manual test creation. The cmdlet provides information about test set configuration, execution order enforcement, coverage settings, and associated test cases.

Primary Endpoint: GET /testmanager_/api/v2/{projectId}/testsets

OAuth required scopes: TM.TestSets or TM.TestSets.Read

Required permissions: TestSet.Read

## EXAMPLES

### Example 1
```powershell
PS Orch1Tm:\> Get-TmTestSet
```

Retrieves all test sets from the current Test Manager project.

### Example 2
```powershell
PS Orch1Tm:\> Get-TmTestSet RegressionTests
```

Retrieves the test set named "RegressionTests" from the current project.

### Example 3
```powershell
PS Orch1Tm:\> Get-TmTestSet Smoke*
```

Retrieves all test sets whose names start with "Smoke" using wildcard matching.

### Example 4
```powershell
PS Orch1Tm:\> Get-TmTestSet -Recurse
```

Retrieves all test sets from all Test Manager projects recursively.

### Example 5
```powershell
PS Orch1Tm:\> Get-TmTestSet -Path Orch1Tm:\MyProject
```

Retrieves all test sets from the specified Test Manager project without changing the current location.

### Example 6
```powershell
PS Orch1Tm:\> Get-TmTestSet -Recurse | Where-Object {$_.source -eq "Orchestrator"}
```

Retrieves all test sets that originate from Orchestrator processes across all projects.

### Example 7
```powershell
PS Orch1Tm:\> Get-TmTestSet -Recurse | ConvertTo-Json -Depth 3
```

Retrieves all test sets and displays their complete structure in JSON format for detailed analysis.

### Example 8
```powershell
PS Orch1Tm:\> Get-TmTestSet | Where-Object {$_.enforceExecutionOrder -eq $true} | Select-Object name, numberOfTestCases
```

Finds test sets that enforce execution order and displays their names and test case counts.

## NOTES

This cmdlet operates on folder entities, which means you may need to navigate to specific Test Manager projects or use the -Path parameter to target the desired location.

For Test Manager operations, ensure you are connected to the appropriate Test Manager drive (Orch1Tm:, Orch2Tm:, etc.) or use the -Path parameter to specify the correct drive.

When using the -Recurse parameter with large Test Manager environments, the operation may take considerable time to complete. Consider using -Path to limit the scope if needed.

Test sets contain important execution configuration such as:
- enforceExecutionOrder: Whether test cases must be executed in a specific order
- enableCoverage: Whether code coverage analysis is enabled
- source: The origin of the test set (Orchestrator, external systems, etc.)

Use ConvertTo-Json to explore the complete structure of test set objects, as they contain detailed metadata that may not be visible in the default display format.

Parameter placement is important for folder entity cmdlets. Place -Path, -Recurse parameters immediately after the cmdlet name for proper auto-completion functionality.
