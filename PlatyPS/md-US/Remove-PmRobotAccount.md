# Remove-PmRobotAccount

## SYNOPSIS
Removes robot accounts from UiPath Platform Management.

## SYNTAX

```
Remove-PmRobotAccount [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Remove-PmRobotAccount cmdlet removes robot accounts from UiPath Platform Management. Robot accounts are service accounts used for unattended automation scenarios. This cmdlet permanently removes the specified robot accounts and their associated configurations.

The -Path parameter specifies target drives. If not specified, the current drive will be targeted.

This cmdlet accesses Platform Management APIs and works across all UiPath Orchestrator drives (Orch1:, Orch1Tm:, Orch1Du:).

Primary Endpoint: DELETE /api/RobotAccount/{tenantId}/{robotAccountId}

OAuth required scopes: PM.RobotAccount

Required permissions: [PLACEHOLDER - Platform Management robot account delete permissions]

## EXAMPLES

### Example 1: Remove multiple robot accounts
```powershell
PS Orch1:\> Remove-PmRobotAccount RobotAccount1, RobotAccount2, RobotAccount3 -WhatIf
```

Removes multiple robot accounts in a single operation.

### Example 2: Remove with confirmation prompt
```powershell
PS Orch1:\> Remove-PmRobotAccount Robot* -Confirm
```

Removes the robot account with an additional confirmation prompt before proceeding.

### Example 3: Remove robot accounts from specific drives
```powershell
PS C:\> Remove-PmRobotAccount -Path Orch1:, Orch2: RobotAccount1
```

Removes the robot account from multiple specified Orchestrator drives.

## NOTES
- This cmdlet permanently removes robot accounts and cannot be undone
- Use -WhatIf parameter to preview the removal operation before executing
- Robot accounts are used for unattended automation scenarios
- Removing a robot account will affect any automation processes that depend on it
- Platform Management cmdlets (starting with "Pm") provide cross-organizational robot account management
- Always verify robot account dependencies before removal to avoid disrupting automation workflows
