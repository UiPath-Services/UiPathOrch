# Remove-OrchMachine

## SYNOPSIS
Removes machines from the Orchestrator.

## SYNTAX

```
Remove-OrchMachine [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Remove-OrchMachine cmdlet permanently removes machines from the Orchestrator environment. Machines represent physical or virtual computers that can host UiPath robots and execute automation processes.

When a machine is removed, all robots associated with that machine are also removed from the Orchestrator. This action cannot be undone, so careful consideration should be taken before removing machines, especially those that are actively used for automation execution.

The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before deletion. It is strongly recommended to use these parameters when removing machines to avoid accidental deletion of critical automation infrastructure.

Primary Endpoint: DELETE /odata/Machines({machineId})

OAuth required scopes: OR.Machines

Required permissions: Machines.Delete

## EXAMPLES

### Example 1
```powershell
PS C:\> Remove-OrchMachine TestMachine
```

Removes the machine named "TestMachine" from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Remove-OrchMachine TestMachine1, TestMachine2 -WhatIf
```

Shows what would happen if the specified machines were removed, without actually performing the removal.

### Example 3
```powershell
PS C:\> Remove-OrchMachine Temp* -Confirm
```

Removes all machines whose names start with "Temp", prompting for confirmation before each removal.

### Example 4
```powershell
PS C:\> Remove-OrchMachine OldMachine -Path Orch1:, Orch2:
```

Removes the machine named "OldMachine" from multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchMachine | Where-Object {$_.Type -eq "Template" -and $_.Name -like "*Test*"} | Remove-OrchMachine -WhatIf
```

Identifies template machines containing "Test" in their name and shows what would be removed without actually performing the deletion.

### Example 6
```powershell
PS C:\> Remove-OrchMachine UnusedMachine -Confirm | Out-Null
```

Removes a machine with confirmation prompt and suppresses the output.

## NOTES

This cmdlet operates on tenant entities, which means it affects machines across the entire Orchestrator tenant rather than specific folders.

Before removing machines, ensure they are not actively running processes or jobs. You can use Get-OrchJob to check for running jobs on specific machines.

Removing a machine also removes all robots associated with that machine. If you need to preserve robot configurations, consider using Copy-OrchMachine to backup the machine configuration before removal.

Template machines and system-generated machines should be removed with extreme caution as they may be required for proper functioning of the Orchestrator environment.

Use the -WhatIf parameter to preview the operation before actual execution, especially when using wildcard patterns that might match multiple machines.

The removal operation is irreversible. Once a machine is deleted, it cannot be recovered and must be recreated if needed.
