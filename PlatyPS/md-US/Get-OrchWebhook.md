# Get-OrchWebhook

## SYNOPSIS
Gets webhooks from UiPath Orchestrator.

## SYNTAX

```
Get-OrchWebhook [[-Name] <String[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchWebhook` cmdlet retrieves webhook information from UiPath Orchestrator. Webhooks are HTTP callbacks that allow Orchestrator to send real-time notifications to external systems when specific events occur, such as job completions, process deployments, or queue item updates.

This cmdlet returns detailed information about configured webhooks including their URLs, event subscriptions, security settings, and operational status. It supports filtering by webhook name using wildcards.

Webhooks enable integration with external monitoring systems, notification platforms, and custom applications by providing real-time event notifications from Orchestrator.

Multiple values for the -Name and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: GET /odata/Webhooks

OAuth required scopes: OR.Webhooks or OR.Webhooks.Read

Required permissions: Webhooks.View

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchWebhook
```

Gets all webhooks configured in the current Orchestrator instance.

### Example 2
```powershell
PS Orch1:\> Get-OrchWebhook my*
```

Gets webhooks whose names start with "my" using wildcard pattern matching.

### Example 3
```powershell
PS C:\> Get-OrchWebhook -Path Orch1:, Orch2: integration*
```

Gets webhooks with names starting with "integration" from multiple tenants.

### Example 4
```powershell
PS Orch1:\> Get-OrchWebhook mywebhook | ConvertTo-Json
```

Gets a specific webhook and displays the complete structure including nested properties like Events array.

## NOTES
- Webhooks provide real-time event notifications from Orchestrator to external systems
- The Events property contains an array of WebhookEvent objects specifying which events trigger the webhook
- Common event types include: job.created, job.faulted, job.completed, process.deleted, queue.created, queue.deleted, and many others
- The SubscribeToAllEvents property indicates whether the webhook receives all available event types
- The AllowInsecureSsl property controls whether the webhook accepts invalid or self-signed SSL certificates
- The Secret property (if configured) is used for webhook signature verification but is not displayed for security
- The Key property contains a unique identifier for the webhook
- Disabled webhooks (Enabled = false) do not send notifications even if events occur
