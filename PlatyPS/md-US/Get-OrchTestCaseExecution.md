# Get-OrchTestCaseExecution

## SYNOPSIS
Gets test case execution records from UiPath Orchestrator.

## SYNTAX

```
Get-OrchTestCaseExecution [[-Name] <String[]>] [-Path <String[]>] [-Recurse] [-Depth <UInt32>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchTestCaseExecution cmdlet retrieves test case execution records from UiPath Orchestrator. Test case executions represent individual runs of automated test cases, including their execution status, timing information, input/output arguments, and related metadata.

This is a folder entity operation. You must either navigate to a specific folder using Set-Location (cd) or use the -Path, -Recurse, or -Depth parameters to specify target folders.

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchTestCaseExecution
```

Retrieves all test case execution records from the current folder.

### Example 2
```powershell
PS Orch1:\> Get-OrchTestCaseExecution -Recurse
```

Retrieves all test case execution records from the current folder and all subfolders recursively.

### Example 3
```powershell
PS Orch1:\Shared> Get-OrchTestCaseExecution -Recurse TestCase*
```

Retrieves test case executions whose names start with "TestCase" from all folders recursively using positional parameter.

### Example 4
```powershell
PS C:\> Get-OrchTestCaseExecution -Path Orch1:\Production
```

Retrieves test case execution records from the Production folder, demonstrating execution from any location.

### Example 5
```powershell
PS Orch1:\Shared> Get-OrchTestCaseExecution | Select-Object -First 1 | ConvertTo-Json -Depth 2
```

Retrieves the first test case execution and displays its structure in JSON format for detailed analysis.

## NOTES
- This cmdlet operates on folder entities and requires folder navigation or path specification
- For optimal performance, place -Path, -Recurse, and -Depth parameters immediately after the cmdlet name to enable proper autocompletion
- Common execution statuses include "Passed", "Failed", "Cancelled", "Pending", and others
- InputArguments and OutputArguments contain JSON-formatted test data
- Use filtering and sorting operations to analyze test execution patterns and results
- The RuntimeType property indicates the type of robot that executed the test (e.g., "TestAutomation")
- ExecutionOrder property shows the sequence of test case execution within a test set
