# Set-OrchAsset

## SYNOPSIS
Creates, updates, and removes the assets.

## SYNTAX

```
Set-OrchAsset [[-ValueType] <String>] [-Name] <String[]> [[-Value] <String>] [[-UserName] <String[]>]
 [[-MachineName] <String[]>] [-Description <String>] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Set-OrchAsset cmdlet creates, updates, and removes assets in UiPath Orchestrator folders. Assets are key-value pairs that store configuration data, credentials, and other information needed by automation processes.

This cmdlet provides a unified interface for asset management - it creates new assets if they don't exist, updates existing assets when specified, and can remove assets when no value is provided. The cmdlet supports different asset types including Text, Bool, Integer, and Credential assets.

Assets can be scoped globally (available to all robots) or per-robot (specific values for individual robots or machines). Use the -UserName and -MachineName parameters to create per-robot assets with specific values for different execution contexts.

This is a folder entity cmdlet. Use Set-Location cmdlet (cd command) to navigate to the target folder first, or specify the target folders using -Path parameters.

Primary Endpoint: POST /odata/Assets, PUT /odata/Assets({asset.Id}), DELETE /odata/Assets({assetId})

OAuth required scopes: OR.Assets

Required permissions: Assets.Create, Assets.Edit, Assets.Delete

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Set-OrchAsset -ValueType Text -Name TestAsset2024 -Value "Hello World" -Description "Test asset created via PowerShell" -WhatIf
```

```
What if: Performing the operation "Add Asset" on target "Orch1:\Shared\TestAsset2024".
```

Demonstrates creating a new text asset with -WhatIf to preview the operation before execution.

### Example 2
```powershell
PS Orch1:\Shared> Set-OrchAsset -ValueType Text -Name TestAsset2024 -Value "Hello World" -Description "Test asset created via PowerShell"
```

Creates a new text asset with the specified name, value, and description.

### Example 3
```powershell
PS Orch1:\Shared> Set-OrchAsset -Name TestAsset2024 -Value "Updated Value 2024"
```

Updates an existing asset with a new value. The ValueType and Description remain unchanged.

### Example 4
```powershell
PS Orch1:\Shared> Set-OrchAsset -ValueType Bool -Name BoolAsset2024 -Value "True" -Description "Boolean test asset"
```

Creates a Boolean asset with a True value.

### Example 5
```powershell
PS Orch1:\Shared> Set-OrchAsset -ValueType Integer -Name CounterAsset -Value "100" -Description "Counter value"
```

Creates an Integer asset with a numeric value.

### Example 6
```powershell
PS C:\> Set-OrchAsset -Path Orch1:\Production -Name ConfigAsset -Value "ProductionConfig" -Description "Production configuration"
```

Creates or updates an asset in the specified Production folder using the -Path parameter.

## NOTES
This cmdlet serves as a unified interface for asset lifecycle management - creating, updating, and removing assets based on the provided parameters. When creating new assets, both ValueType and Value parameters are typically required. For updates, only the Name and new Value are needed. The cmdlet automatically determines whether to create or update based on asset existence.
