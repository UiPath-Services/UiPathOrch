# Search-PmDirectory

## SYNOPSIS
Searches directory users and groups in Platform Management.

## SYNTAX

```
Search-PmDirectory [-Name] <String> [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Search-PmDirectory` cmdlet searches for directory users and groups in UiPath Platform Management based on a name pattern. This cmdlet provides a way to find users, groups, and robot users across the directory by searching for names that start with the specified search term.

The search results include comprehensive information about directory objects such as users (DirectoryUser), groups (DirectoryGroup), and robot users (DirectoryRobotUser), along with their identifiers, display names, email addresses, and source information.

This is a Platform Management API cmdlet that works on any UiPathOrch drive (Orch1:, Orch1Tm:, Orch1Du:) and provides essential functionality for user and group discovery in automation workflows and administrative tasks.

The search is performed using a "starts with" pattern, making it efficient for finding directory objects when you know the beginning of the name or identifier.

Primary Endpoint: GET /api/Directory/Search/{partitionGlobalId}?startsWith={userName}

## EXAMPLES

### Example 1
```powershell
PS Orch1:\> Search-PmDirectory y
```

Searches for all directory objects (users, groups, robot users) whose names start with "y".

### Example 2
```powershell
PS Orch1:\> Search-PmDirectory user@example.com
```

Searches for directory objects with the exact name or starting with "user@example.com".

### Example 3
```powershell
PS Orch1:\> Search-PmDirectory admin | Where-Object objectType -eq "DirectoryGroup"
```

Searches for directory objects starting with "admin" and filters to show only groups.

### Example 4
```powershell
PS Orch1:\> $result = Search-PmDirectory user@example.com
PS Orch1:\> $result | ConvertTo-Json -Depth 3
```

Searches for directory objects and displays the complete structure in JSON format to explore all properties.

### Example 5
```powershell
PS Orch1:\> Search-PmDirectory y | Select-Object identityName, displayName, objectType
```

Searches for directory objects starting with "y" and displays key properties.

## NOTES
- This cmdlet uses Platform Management API and works on any UiPathOrch drive
- The search uses a "starts with" pattern rather than exact matching
- Object types include DirectoryUser (individual users), DirectoryGroup (groups), and DirectoryRobotUser (robot accounts)
- The type property contains numeric values corresponding to the object types
- The source property indicates the origin of the directory object (typically "local")
- Use ConvertTo-Json to explore the complete object structure and understand all available properties
- This cmdlet is useful for user/group discovery in automation scripts and administrative tasks
