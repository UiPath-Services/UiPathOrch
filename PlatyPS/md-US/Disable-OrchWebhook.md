# Disable-OrchWebhook

## SYNOPSIS
Disables webhooks in the Orchestrator.

## SYNTAX

```
Disable-OrchWebhook [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Disable-OrchWebhook cmdlet disables previously enabled webhooks in the Orchestrator environment. When a webhook is disabled, it will stop sending HTTP notifications to external systems, effectively pausing the integration without removing the webhook configuration.

Disabling webhooks is useful for temporarily stopping notifications during maintenance periods, troubleshooting integration issues, or when external systems are unavailable. The webhook configuration remains intact and can be re-enabled later using the Enable-OrchWebhook cmdlet.

The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before disabling webhooks.

Primary Endpoint: PATCH /odata/Webhooks({webhookId})

OAuth required scopes: OR.Webhooks

Required permissions: Webhooks.Edit

## EXAMPLES

### Example 1
```powershell
PS C:\> Disable-OrchWebhook IntegrationHook
```

Disables the webhook named "IntegrationHook".

### Example 2
```powershell
PS C:\> Disable-OrchWebhook Hook1, Hook2
```

Disables multiple webhooks by specifying their names in a comma-separated list.

### Example 3
```powershell
PS C:\> Disable-OrchWebhook Test* -WhatIf
```

Shows what would happen if all webhooks whose names start with "Test" were disabled, without actually performing the operation.

### Example 4
```powershell
PS C:\> Disable-OrchWebhook MaintenanceHook -Path Orch1:, Orch2:
```

Disables the webhook named "MaintenanceHook" in multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchWebhook | Where-Object Enabled -eq $true | Disable-OrchWebhook -Confirm
```

Finds all enabled webhooks and disables them with confirmation prompts for each operation.

### Example 6
```powershell
PS C:\> Disable-OrchWebhook CriticalEventsHook | Select-Object Name, Enabled, Url
```

Disables a webhook and displays its updated status including name, enabled state, and URL.

## NOTES

This cmdlet operates on tenant entities, which means it affects webhooks across the entire Orchestrator tenant rather than specific folders.

Once disabled, webhooks will immediately stop sending notifications. Any events that occur while the webhook is disabled will not be sent to the target URL.

Use Get-OrchWebhook to verify the current status of webhooks before and after disabling them.

Consider using -WhatIf to preview the operation, especially when using wildcard patterns that might match multiple webhooks.

Disabling webhooks is a reversible operation. Use Enable-OrchWebhook to re-enable webhooks when needed.

Temporary disabling of webhooks is preferable to removal when you plan to use the same webhook configuration again in the future.
