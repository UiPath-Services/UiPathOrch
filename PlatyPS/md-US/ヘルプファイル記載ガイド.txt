# UiPathOrch Module ヘルプファイル記載ガイド

## ファイル構成と作業の進め方

### 英語版ヘルプのソース
`
C:\MyProj\OrchProvider\PlatyPS\md-US
`

### 作業の原則
- **品質を最優先**: 時間的効率より品質重視
- **実動作確認必須**: すべての例を実際に実行して動作確認
- **英語版を先に完了**: その後で日本語版を記述

### 動作確認の手法
1. **実際の実行**: cmdletを実行し動作確認
2. **JSON出力確認**: ConvertTo-Jsonで詳細構造を確認
3. **複数パラメータ検証**: 異なる組み合わせをテスト

## **CRITICAL: 実動作確認の必須化**

### すべての例は実際に実行して確認
- -WhatIfで安全確認
- パラメータ構造の確認: Get-Help <cmdlet> | Select-Object -ExpandProperty syntax
- **検証必須**: 記載したコマンドが実際に動作する

## **CRITICAL: ファイル整合性チェックの必須化**

### ファイル編集前の整合性確認
```powershell
# ファイル名とヘッダーの一致確認
$fileName = "Get-OrchProcess" # 編集対象のcmdlet名
$content = Get-Content "md-US\$fileName.md" | Where-Object { $_ -match "^# " } | Select-Object -First 1
$headerName = $content -replace "^# ", ""
if ($fileName -ne $headerName) {
    Write-Warning "❌ ファイル破損検出: $fileName.md のヘッダーが '$headerName' になっています"
    # この場合は git checkout HEAD -- "md-US\$fileName.md" で復元
}
```

### ファイル編集後の整合性確認
```powershell
# 編集完了後に必ず実行
Update-MarkdownHelp -Path "md-US\$fileName.md" -Encoding ([System.Text.Encoding]::UTF8)
# エラーが出た場合は構文に問題あり
```

### 一括整合性チェックスクリプト
```powershell
# 全ファイルの整合性チェック
Get-ChildItem md-US -Filter "*.md" | ForEach-Object {
    $fileName = $_.BaseName
    $content = Get-Content $_.FullName | Where-Object { $_ -match "^# " } | Select-Object -First 1
    if ($content) {
        $headerName = $content -replace "^# ", ""
        if ($fileName -ne $headerName -and $fileName -ne "about_UiPathOrch" -and $fileName -ne "UiPathOrch") {
            Write-Warning "❌ 破損ファイル: $($_.Name) (ヘッダー: $headerName)"
        }
    }
}
```

## エンティティの種類と特徴

### テナントエンティティ vs フォルダエンティティ

#### フォルダエンティティの特徴
- エラーメッセージ: Use Set-Location cmdlet (cd command) to navigate to the target folder first, or specify the target folders using -Path, -Recurse, or -Depth parameters
- **必要な操作**: フォルダに移動するか、-Path/-Recurse/-Depthを指定
- **-Recurseパラメータ**をもつ

#### テナントエンティティの特徴
- **-Pathの意味**: ドライブ名の指定
- **例**: Get-OrchLibrary -Path Orch1:, Orch2:

## **パラメータ順序の重要原則**

### **-Path と -Recurse の優先配置**
フォルダエンティティでは、**-Path、-Recurse、-Depth パラメータを他のパラメータより先に指定**：

**✅ 適切な例:**
`powershell
Get-OrchAsset -Path Orch1:\Production AssetName
Get-OrchProcess -Recurse ProcessName  
Enable-OrchFolderMachineInherit -Path Orch1:\Development Machine1, Machine2 -WhatIf
`

**理由**: PowerShellのIntelliSense（Ctrl+Space）が適切に動作

## **Positional Parameter の優先使用**

パラメータ名を省略した簡潔な形式を優先：

**✅ 簡潔で実用的:**
`powershell
Remove-OrchWebhook TestWebhook
Copy-OrchRole CustomRole Orch2:
`

**❌ 冗長:**
`powershell
Remove-OrchWebhook -Name TestWebhook
Copy-OrchRole -Name CustomRole -Destination Orch2:
`


## ワイルドカード制約（重要）

**ドライブ名**: ワイルドカード使用不可
- ❌ Get-OrchUpdateSetting -Path Orch*
- ✅ Get-OrchUpdateSetting -Path Orch1:, Orch2:

**フォルダ名**: ワイルドカード使用可能
- ✅ Get-OrchAsset -Path Orch1:\Production *Data*

※PSProvider共通の制約（UiPathOrch固有ではない）
## ダブルクォート使用ルール

**✅ 必要な場合:**
- スペース含む: "My Process Name"
- ファイルパス: "C:\Program Files\logs.txt"

**❌ 不要な場合:**
- 単純文字列: TestRole (not "TestRole")
- ワイルドカード: *Auth* (not "*Auth*")

### **Where-Object vs パラメータフィルタ**
コマンドレットにフィルタパラメータがある場合は、Where-Objectではなくそのパラメータを使用：

**✅ 良い例:**
`powershell
Get-OrchLibrary -HostFeed -Id *Excel*
Get-OrchSetting -Name *Auth*
`

**❌ 悪い例:**
`powershell
Get-OrchLibrary -HostFeed | Where-Object {.Id -like "*Excel*"}
`


## **破損ファイル予防策**

### **禁止事項**
1. **ファイルコピーによるテンプレート作成禁止**
   - 既存の .md ファイルをコピーして新しいcmdletのベースにしない
   - 必ずコマンドからの新規生成を使用

2. **ヘッダー編集時の注意**
   - `# cmdlet名` のヘッダーはファイル名と完全一致させる
   - 編集後は必ず整合性チェックを実行

3. **一括編集時の注意**
   - 複数ファイルを同時編集する際は、各ファイルで整合性チェック
   - 類似名のcmdletでは特に注意が必要

### **推奨手順**
1. **編集前**: 必ず整合性チェック実行
2. **編集中**: ヘッダー行は慎重に扱う
3. **編集後**: 構文検証と整合性チェック両方実行

## **UiPathOrch パイプライン入力の重要原則**

### **ByPropertyName Binding**
すべてのUiPathOrch cmdletパラメータは `ValueFromPipelineByPropertyName = true`

**✅ 推奨（簡潔で効率的）:**
```powershell
Get-OrchRole | Where-Object {$_.IsStatic -eq $false} | Copy-OrchRole -Destination Orch2:
Get-OrchMachine | Where-Object {$_.Type -eq "Template"} | Copy-OrchMachine -Destination Orch2:
```

**❌ 非推奨（冗長で非効率）:**
```powershell
Get-OrchRole | Where-Object {$_.IsStatic -eq $false} | ForEach-Object { Copy-OrchRole $_.Name Orch2: }
```

### **INPUTSセクション記載**
Entity オブジェクトのパイプライン入力を必ず記載：
```markdown
### System.String[]
Names can be piped to this cmdlet.

### UiPath.PowerShell.Entities.[EntityType]
Entity objects can be piped to this cmdlet. The Name property will be automatically mapped to the -Name parameter via ByPropertyName binding.
```
