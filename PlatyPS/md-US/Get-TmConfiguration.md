# Get-TmConfiguration

## SYNOPSIS
Gets the Test Manager configuration settings.

## SYNTAX

```
Get-TmConfiguration [[-Path] <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-TmConfiguration` cmdlet retrieves the configuration settings for UiPath Test Manager. This includes endpoint configurations, basic settings, identity server configuration, storage configuration, search configuration, platform configuration, and various third-party integrations such as Azure DevOps, Jira, ServiceNow, and others.

The configuration object contains detailed information about how Test Manager is configured to integrate with external systems and manage test automation workflows.

Multiple values for the -Path parameter can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

## EXAMPLES

### Example 1
```powershell
PS C:\> Set-Location Orch1Tm:
PS Orch1Tm:\> Get-TmConfiguration
```

Gets the complete Test Manager configuration for the current Test Manager instance.

### Example 2
```powershell
PS Orch1Tm:\> $config = Get-TmConfiguration
PS Orch1Tm:\> $config.basicConfiguration
```

Retrieves the Test Manager configuration and then displays only the basic configuration settings.

### Example 3
```powershell
PS Orch1Tm:\> Get-TmConfiguration | Select-Object endpoints, platformConfiguration
```

Gets the Test Manager configuration and displays only the endpoints and platform configuration properties.

## NOTES
- This cmdlet requires access to a Test Manager instance
- The configuration object contains settings for various integrations including Azure DevOps, Jira (Server, Cloud Basic Auth, Cloud OAuth), ServiceNow, WebHook, XRay, Redmine, qTest, and SAP
- Some configuration properties may be null if the corresponding integrations are not configured
