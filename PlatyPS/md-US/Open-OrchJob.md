# Open-OrchJob

## SYNOPSIS
Opens job details in the default web browser.

## SYNTAX

```
Open-OrchJob [[-Id] <Int64[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Open-OrchJob` cmdlet opens the job details page in the default web browser for the specified job ID(s). This provides a convenient way to view detailed job information, execution logs, and other job-related data directly in the Orchestrator web interface.

This cmdlet is useful for quickly navigating from PowerShell to the web interface to investigate job details, view execution logs, or access features not available through the command line.

Multiple job IDs can be specified to open multiple job pages simultaneously. You can use autocomplete for job IDs by pressing [Ctrl+Space] or [Tab].

## EXAMPLES

### Example 1
```powershell
PS C:\> Set-Location Orch1:\MyWorkspace
PS Orch1:\MyWorkspace> Open-OrchJob -Id 108984122
```

Opens the job details page for job ID 108984122 in the default web browser.

### Example 2
```powershell
PS Orch1:\MyWorkspace> $job = Get-OrchJob -First 1
PS Orch1:\MyWorkspace> Open-OrchJob -Id $job.Id
```

Gets the first job and opens its details page in the browser.

### Example 3
```powershell
PS Orch1:\> Open-OrchJob -Id 108984122,108983813
```

Opens job details pages for multiple jobs in separate browser tabs or windows.

### Example 4
```powershell
PS Orch1:\> Get-OrchJob -State "Faulted" -First 5 | Open-OrchJob
```

Gets the first 5 faulted jobs and opens their details pages for investigation.

### Example 5
```powershell
PS Orch1:\MyWorkspace> Get-OrchJob -Last "1h" | Where-Object State -eq "Faulted" | Open-OrchJob
```

Opens job details pages for all jobs that failed in the last hour.

## NOTES

- The cmdlet requires an active web browser and internet connectivity to the Orchestrator instance

- The user must have appropriate permissions to view job details in the web interface
