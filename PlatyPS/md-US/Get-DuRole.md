# Get-DuRole

## SYNOPSIS
Gets Document Understanding roles from UiPath Orchestrator.

## SYNTAX

```
Get-DuRole [[-Name] <String[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-DuRole` cmdlet retrieves Document Understanding role information from UiPath Orchestrator. Document Understanding roles define the permissions and access levels for users working with document processing projects, classifiers, extractors, and document types.

This cmdlet returns comprehensive role information including role descriptions, creation details, permission types (BUILTIN vs custom), and detailed action permissions. Each role contains specific actionDetails that define granular permissions for various Document Understanding operations such as reading projects, managing classifiers, updating document types, and monitoring project performance.

The cmdlet operates on Document Understanding drives (Orch1Du:) and supports filtering by role name using wildcards.

Multiple values for the -Name and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

## EXAMPLES

### Example 1
```powershell
PS C:\> cd Orch1Du:\
PS Orch1Du:\> Get-DuRole
```

Gets all Document Understanding roles available in the current instance.

### Example 2
```powershell
PS Orch1Du:\> Get-DuRole *developer*
```

Gets Document Understanding roles whose names contain "developer".

### Example 3
```powershell
PS Orch1Du:\> $role = Get-DuRole "DU Data Annotator"
PS Orch1Du:\> $role.actionDetails | Select-Object name, description | Format-Table
```

Gets the DU Data Annotator role and displays its specific action permissions in a formatted table.

### Example 4
```powershell
PS Orch1Du:\> Get-DuRole | Where-Object type -eq "BUILTIN"
```

Gets only built-in Document Understanding roles (as opposed to custom roles).

### Example 5
```powershell
PS Orch1Du:\> $role = Get-DuRole | Select-Object -First 1
PS Orch1Du:\> $role | ConvertTo-Json -Depth 3
```

Gets the first role and displays its complete structure in JSON format to explore all properties including detailed actionDetails.

## NOTES
- This cmdlet operates on Document Understanding drives (Orch1Du:)
- Common built-in roles include: DU Data Annotator, DU Developer, DU Model Trainer, DU Project Administrator, and DU Viewer
- The actionDetails property contains granular permissions for specific Document Understanding operations
- Role types include BUILTIN (predefined system roles) and custom roles created by administrators
- Each action detail specifies namespace (DOCUMENTUNDERSTANDING), resource actions (Read, Update, Create, Delete), and resource groups
- Use ConvertTo-Json to explore the complete actionDetails structure and understand specific permissions
