# Copy-OrchMachine

## SYNOPSIS
Copies machine templates between UiPath Orchestrator tenants.

## SYNTAX

```
Copy-OrchMachine [-Name] <String[]> [-Destination] <String[]> [-Path <String>]
 [-ProgressAction <ActionPreference>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The `Copy-OrchMachine` cmdlet copies machine templates from the current UiPath Orchestrator tenant to one or more destination tenants. This is useful for replicating machine configurations across multiple Orchestrator environments or when setting up new tenants with standardized machine templates.

The cmdlet operates on tenant entities, copying machine templates including their configuration settings such as slots allocation, scope, automation type, and other properties. Note that sensitive information like license keys and client secrets are regenerated for the destination tenant.

Primary Endpoint: GET /odata/Machines, POST /odata/Machines

OAuth required scopes: OR.Machines

Required permissions: Machines.View, Machines.Create

## EXAMPLES

### Example 1
```powershell
PS Orch1:\> Copy-OrchMachine "[Default] Agent Service" Orch2:
```

Copies the "[Default] Agent Service" machine template to the Orch2 tenant.

### Example 2
```powershell
PS Orch1:\> Copy-OrchMachine "*Template*" Orch2:, Orch3: -WhatIf
```

Shows what would happen when copying all machine templates with "Template" in their name to both Orch2 and Orch3 tenants, without actually performing the copy operation.

### Example 3
```powershell
PS Orch1:\> Get-OrchMachine | Where-Object {$_.Type -eq "Template"} | Copy-OrchMachine Orch2:
```

Gets all machine templates and copies them to Orch2 tenant using pipeline input.

### Example 4
```powershell
PS Orch1:\> Copy-OrchMachine "Production Template" Orch2: -Confirm
```

Copies the "Production Template" machine to Orch2 with confirmation prompt before execution.

### Example 5
```powershell
PS C:\> Copy-OrchMachine -Path Orch1: "Cloud Robot Template" Orch2:, Orch3:
```

Copies the "Cloud Robot Template" from Orch1 tenant to both Orch2 and Orch3 tenants, specifying the source tenant explicitly.

## NOTES

**Important Security Considerations:**
- License keys and client secrets are regenerated in the destination tenant for security reasons
- The copied machine templates will have new unique identifiers
- Ensure you have appropriate permissions in both source and destination tenants

**Machine Template Properties Copied:**
- Name and Description
- Machine Type and Scope
- Slot allocations (Unattended, NonProduction, TestAutomation, etc.)
- Automation Type and Target Framework settings
- Update Policy settings
- Tags and maintenance window configurations

**Use Cases:**
- Standardizing machine configurations across multiple tenants
- Setting up new Orchestrator environments with existing templates
- Migrating machine templates between development and production environments
- Creating backup copies of critical machine configurations

**Best Practices:**
- Use -WhatIf to preview changes before execution
- Use -Confirm for critical production operations
- Verify machine template functionality in destination tenant after copying
- Update any tenant-specific configurations after copying

Use Get-OrchMachine to view available machine templates before copying.
