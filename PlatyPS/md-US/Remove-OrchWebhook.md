# Remove-OrchWebhook

## SYNOPSIS
Removes webhooks from the Orchestrator.

## SYNTAX

```
Remove-OrchWebhook [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Remove-OrchWebhook cmdlet permanently removes webhooks from the Orchestrator environment. Webhooks provide HTTP notifications to external systems when specific events occur in Orchestrator.

When a webhook is removed, all its configuration including URL, event types, and custom headers are permanently deleted. This action cannot be undone, so careful consideration should be taken before removing webhooks, especially those that are actively used for critical integrations.

The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before deletion. It is strongly recommended to use these parameters when removing webhooks to avoid accidental deletion of important integration configurations.

Primary Endpoint: GET /odata/Webhooks, DELETE /odata/Webhooks({webhookId})

OAuth required scopes: OR.Webhooks

Required permissions: Webhooks.Delete

## EXAMPLES

### Example 1
```powershell
PS C:\> Remove-OrchWebhook TestWebhook
```

Removes the webhook named "TestWebhook" from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Remove-OrchWebhook TestWebhook1, TestWebhook2 -WhatIf
```

Shows what would happen if the specified webhooks were removed, without actually performing the removal.

### Example 3
```powershell
PS C:\> Remove-OrchWebhook Temp* -Confirm
```

Removes all webhooks whose names start with "Temp", prompting for confirmation before each removal.

### Example 4
```powershell
PS C:\> Remove-OrchWebhook OldIntegration -Path Orch1:, Orch2:
```

Removes the webhook named "OldIntegration" from multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchWebhook | Where-Object {$_.Enabled -eq $false -and $_.Name -like "*test*"} | Remove-OrchWebhook -WhatIf
```

Identifies disabled webhooks containing "test" in their name and shows what would be removed without actually performing the deletion.

### Example 6
```powershell
PS C:\> Remove-OrchWebhook UnusedWebhook -Confirm | Out-Null
```

Removes a webhook with confirmation prompt and suppresses the output.

## NOTES

This cmdlet operates on tenant entities, which means it affects webhooks across the entire Orchestrator tenant rather than specific folders.

Before removing webhooks, consider disabling them first using Disable-OrchWebhook to test the impact on your integrations. This allows you to verify that the removal won't break critical business processes.

If you need to preserve webhook configurations for future use, consider documenting the webhook settings or exporting the configuration before removal.

Use the -WhatIf parameter to preview the operation before actual execution, especially when using wildcard patterns that might match multiple webhooks.

The removal operation is irreversible. Once a webhook is deleted, it cannot be recovered and must be recreated with all its configuration if needed.

Removing webhooks may impact external systems that depend on Orchestrator notifications. Ensure that dependent systems are updated or have alternative notification mechanisms before removing webhooks.
