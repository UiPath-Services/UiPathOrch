# Remove-OrchCalendar

## SYNOPSIS
Removes calendars from the Orchestrator.

## SYNTAX

```
Remove-OrchCalendar [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Remove-OrchCalendar cmdlet permanently removes calendars from the Orchestrator environment. Calendars define business schedules and time zones for automation processes and can include business days, holidays, and specific time periods.

When a calendar is removed, all its associated schedule definitions, business days, holidays, and time zone settings are permanently deleted. This action cannot be undone, so careful consideration should be taken before removing calendars, especially those that are actively used by triggers or scheduled processes.

The cmdlet supports safety features like -WhatIf to preview the operation and -Confirm to request confirmation before deletion. It is strongly recommended to use these parameters when removing calendars to avoid accidental deletion of critical scheduling configurations.

Primary Endpoint: DELETE /odata/Calendars({calendarId})

OAuth required scopes: OR.Settings

Required permissions: Settings.Delete

## EXAMPLES

### Example 1
```powershell
PS C:\> Remove-OrchCalendar TestCalendar
```

Removes the calendar named "TestCalendar" from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Remove-OrchCalendar Calendar1, Calendar2 -WhatIf
```

Shows what would happen if the specified calendars were removed, without actually performing the removal.

### Example 3
```powershell
PS C:\> Remove-OrchCalendar Temp* -Confirm
```

Removes all calendars whose names start with "Temp", prompting for confirmation before each removal.

### Example 4
```powershell
PS C:\> Remove-OrchCalendar OldCalendar -Path Orch1:, Orch2:
```

Removes the calendar named "OldCalendar" from multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchCalendar | Where-Object {$_.Name -like "*test*"} | Remove-OrchCalendar -WhatIf
```

Identifies calendars containing "test" in their name and shows what would be removed without actually performing the deletion.

### Example 6
```powershell
PS C:\> Remove-OrchCalendar UnusedCalendar -Confirm | Out-Null
```

Removes a calendar with confirmation prompt and suppresses the output.

## NOTES

This cmdlet operates on tenant entities, which means it affects calendars across the entire Orchestrator tenant rather than specific folders.

Before removing calendars, ensure they are not being used by active triggers, schedules, or automation processes. You can use Get-OrchTrigger to check for triggers that may be using specific calendars.

Removing a calendar also removes all associated calendar dates, business days, and holiday definitions. If you need to preserve this information, consider copying the calendar configuration before removal.

Use the -WhatIf parameter to preview the operation before actual execution, especially when using wildcard patterns that might match multiple calendars.

The removal operation is irreversible. Once a calendar is deleted, it cannot be recovered and must be recreated with all its schedule definitions if needed.

Calendars that are actively referenced by triggers or scheduled processes should not be removed as this may cause scheduling failures.
