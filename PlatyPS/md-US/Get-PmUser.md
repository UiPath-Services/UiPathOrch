# Get-PmUser

## SYNOPSIS
Gets users from Platform Management.

## SYNTAX

```
Get-PmUser [[-Email] <String[]>] [-Path <String[]>] [-ExportCsv <String>] [-CsvEncoding <Encoding>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-PmUser cmdlet retrieves user information from the UiPath Platform Management service. This cmdlet accesses user data across the entire platform organization, including user accounts, email addresses, display names, group memberships, and account status.

Platform Management users are organization-level entities that can be assigned to multiple tenants and have access to various UiPath services. This cmdlet provides comprehensive user information including their group memberships, account status, and user types.

The cmdlet supports exporting user data to CSV format, which can be useful for user management, auditing, and integration with external systems.

Primary Endpoint: GET /api/User/users/{partitionGlobalId}

OAuth required scopes: PM.User or PM.User.Read

Required permissions: Platform Management user read permissions

## EXAMPLES

### Example 1
```powershell
PS Orch1:\> Get-PmUser
```

Retrieves all users from the Platform Management service.

### Example 2
```powershell
PS C:\> Get-PmUser john.doe@uipath.com
```

Retrieves the specific user with the email address "john.doe@uipath.com".

### Example 3
```powershell
PS C:\> Get-PmUser *@uipath.com
```

Retrieves all users whose email addresses belong to the "company.com" domain using wildcard matching.

### Example 4
```powershell
PS C:\> Get-PmUser -Path Orch1:, Orch2:
```

Retrieves users from multiple Orchestrator environments (Orch1 and Orch2).

### Example 5
```powershell
PS C:\> Get-PmUser | Where-Object {$_.isActive -eq $true} | Select-Object userName, displayName, lastLoginTime
```

Retrieves only active users and displays their username, display name, and last login time.

### Example 6
```powershell
PS C:\> Get-PmUser | ConvertTo-Json -Depth 3
```

Retrieves all users and displays their complete structure in JSON format for detailed analysis.

### Example 7
```powershell
PS C:\> Get-PmUser -ExportCsv users.csv
```

Exports all Platform Management users to a CSV file that can be used for user management or import operations.

## NOTES

This cmdlet accesses Platform Management APIs, which work across all drive types (Orch1:, Orch1Tm:, Orch1Du:) since they operate at the organization level rather than tenant level.

The -ExportCsv parameter produces CSV files with UTF-8 BOM encoding by default, ensuring compatibility with Excel and other applications without character encoding issues.

Use ConvertTo-Json to explore the complete structure of user objects, as they contain nested arrays like groupIDs that may not be visible in the default display format.

For user management operations, the exported CSV can be modified and used with corresponding import or update commands.

When working with large numbers of users, consider using Where-Object to filter results based on specific criteria like active status or group membership.
