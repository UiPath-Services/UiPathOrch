# Get-OrchCurrentUser

## SYNOPSIS
Gets the user connected to the tenant of the target drive.

## SYNTAX

```
Get-OrchCurrentUser [[-Path] <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchCurrentUser` cmdlet retrieves information about the currently authenticated user for the specified Orchestrator drive. This includes user details such as ID, username, full name, email address, user type, and assigned roles.

This cmdlet can only be used for drives connected with non-confidential application settings (user-scoped authentication). It cannot be used with confidential application drives (app-scoped authentication).

The current user may be different for each drive when multiple Orchestrator instances are configured with different authentication contexts.

Multiple values for the -Path parameter can be specified using comma-separated text. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: /odata/Users/UiPath.Server.Configuration.OData.GetCurrentUser

OAuth required scopes: OR.Users or OR.Users.Read

Required permissions: [PLACEHOLDER - requires verification]

## EXAMPLES

### Example 1
```powershell
PS C:\> cd Orch1:\
PS Orch1:\> Get-OrchCurrentUser
```

Gets the current user information for the current Orchestrator drive.

### Example 2
```powershell
PS C:\> Get-OrchCurrentUser -Path Orch1:,Orch2:
```

Gets the current user information for multiple Orchestrator drives to compare authentication contexts.

### Example 3
```powershell
PS Orch1:\> $user = Get-OrchCurrentUser
PS Orch1:\> Write-Host "Current user: $($user.FullName) ($($user.EmailAddress))"
PS Orch1:\> Write-Host "Roles: $($user.RolesList -join ', ')"
```

Retrieves the current user and displays formatted information about the user and their roles.

### Example 4
```powershell
PS Orch1:\> Get-OrchCurrentUser | Select-Object UserName, FullName, EmailAddress, Type, RolesList
```

Gets the current user and displays only the key properties.

### Example 5
```powershell
PS Orch1:\> Get-OrchCurrentUser | Format-List
```

Gets the current user information and displays it in detailed list format.

## NOTES
- This cmdlet only works with non-confidential application connections (user-scoped authentication)
- Confidential application drives (app-scoped authentication) do not support this operation
- The current user may be different for each configured drive
- The RolesList property contains all roles assigned to the current user
- The Type property indicates the user account type (e.g., DirectoryUser)
