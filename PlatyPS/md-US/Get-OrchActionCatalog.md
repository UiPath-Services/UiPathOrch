# Get-OrchActionCatalog

## SYNOPSIS
Gets action catalogs from the Orchestrator.

## SYNTAX

```
Get-OrchActionCatalog [[-Name] <String[]>] [-Path <String[]>] [-Recurse] [-Depth <UInt32>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchActionCatalog cmdlet retrieves action catalogs from the Orchestrator environment. Action catalogs are repositories that store and organize automation actions, workflows, and task definitions for process mining and task mining scenarios.

Action catalogs provide structured storage for reusable automation components and enable centralized management of action libraries. They support features like encryption, retention policies, and hierarchical organization through folders.

This cmdlet operates on folder entities and supports recursive traversal to find action catalogs across different folder hierarchies. Each action catalog includes metadata such as retention policies, encryption status, and folder organization.

Primary Endpoint: GET /odata/TaskCatalogs

OAuth required scopes: OR.Tasks or OR.Tasks.Read

Required permissions: [PLACEHOLDER - Required permissions to be documented]

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchActionCatalog
```

Retrieves all action catalogs from the current folder.

### Example 2
```powershell
PS Orch1:\> Get-OrchActionCatalog -Recurse
```

Retrieves all action catalogs from all folders recursively.

### Example 3
```powershell
PS C:\> Get-OrchActionCatalog -Path Orch1:\Shared
```

Retrieves action catalogs from the Shared folder using an absolute path.

### Example 4
```powershell
PS Orch1:\> Get-OrchActionCatalog -Recurse -Name *catalog
```

Retrieves action catalogs whose names end with "catalog" from all folders.

### Example 5
```powershell
PS C:\> Get-OrchActionCatalog -Path Orch1:\Shared,Orch1:\Production -Recurse
```

Gets action catalogs from Shared and Production folders recursively.

## NOTES

This cmdlet operates on folder entities, which means you may need to navigate to specific folders or use the -Path parameter to target the desired location.

When using the -Recurse parameter with large folder hierarchies, the operation may take considerable time to complete. Consider using -Depth to limit the traversal scope if needed.

Action catalogs support important features including:
- Encryption for sensitive automation components
- Retention policies with configurable actions (Delete, Archive)
- Hierarchical organization through folder structures
- Centralized management of reusable automation actions

Parameter placement is important for folder entity cmdlets. Place -Path, -Recurse, -Depth parameters immediately after the cmdlet name for proper auto-completion functionality.

Use ConvertTo-Json to explore the complete structure of action catalog objects, as they contain detailed metadata including retention settings and organizational information.
