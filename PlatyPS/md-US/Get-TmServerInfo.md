# Get-TmServerInfo

## SYNOPSIS
Gets the server information of Test Manager.

## SYNTAX

```
Get-TmServerInfo [-Path <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-TmServerInfo` cmdlet retrieves server information from UiPath Test Manager, including the version, server type, and operational status. This information is useful for verifying connectivity, checking server health, and confirming the Test Manager version.

This cmdlet operates on the PSDrive of the UiPathOrchTm provider. If the scope in the configuration file includes "TM.", the PSDrive of the UiPathOrchTm provider will be automatically added. You can confirm this with the Get-PSDrive cmdlet. The configuration file can be opened with the Edit-OrchConfig cmdlet.

Multiple values for the -Path parameter can be specified using comma-separated text. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

## EXAMPLES

### Example 1
```powershell
PS Orch1Tm:\> Get-TmServerInfo
```

Gets the server information for the current Test Manager instance.

### Example 2
```powershell
PS C:\> Get-TmServerInfo -Path Orch1Tm:,Orch2Tm:
```

Gets server information from multiple Test Manager instances for comparison.

### Example 3
```powershell
PS Orch1Tm:\> Get-TmServerInfo | ConvertTo-Json
```

Gets server information and converts the output to JSON format for detailed analysis.

## NOTES
- This cmdlet requires access to a Test Manager instance through the UiPathOrchTm provider
- The UiPathOrchTm PSDrive is automatically added when "TM." scope is included in the configuration
