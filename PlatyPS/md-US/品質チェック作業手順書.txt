# 【新規追加】存在しないフォルダ/エンティティでのExample検証手順

Write-Host "=== 存在しないパス検証 ===" -ForegroundColor Yellow
try {
    # ヘルプ記載のExampleをそのまま実行
    Get-OrchQueue -Path Orch1:\Finance -Recurse

    Write-Host "予想通りエラー: $($_.Exception.Message)" -ForegroundColor Red
    
    # 2. 存在するフォルダでの代替実行
    Write-Host "=== 代替フォルダでの機能検証 ===" -ForegroundColor Yellow
    
    # 実在するフォルダを確認
    $existingFolders = Get-OrchQueue -Recurse | Select-Object -ExpandProperty Path -Unique | Sort-Object
    Write-Host "利用可能なフォルダ:"
    $existingFolders | Select-Object -First 5 | ForEach-Object { Write-Host "  $_" -ForegroundColor Green }
    
    # 3. 同等の機能を実在フォルダで検証
    Write-Host "同等機能の検証:" -ForegroundColor Cyan
    $testFolders = $existingFolders | Select-Object -First 2
    
    if ($testFolders.Count -ge 2) {
        $result = Get-OrchQueue -Path $testFolders[0], $testFolders[1] -Recurse
        Write-Host "✅ 複数パス指定機能確認: $($result.Count) 件取得" -ForegroundColor Green
        
        # パス別分析
        $result | Group-Object Path | ForEach-Object {
            Write-Host "  $($_.Name): $($_.Count)件" -ForegroundColor Gray
        }
    }
}

#### 3.14 **🎯 -Pathパラメータプロンプト設計原則【CRITICAL】**
**重要**: パラメータの真の価値を示すプロンプト設計

`markdown
| パラメータ使用 | 適切なプロンプト | 理由 | Example |
|----------------|------------------|------|---------|
| **パラメータなし** | PS Orch1:\> | テナント/フォルダ内実行 | PS Orch1:\> Get-OrchPersonalWorkspace |
| **-Path指定** | PS C:\> | 任意場所実行を強調 | PS C:\> Get-OrchPersonalWorkspace -Path Orch1:, Orch2: |
| **-Recurse等** | PS C:\> | 外部からの指定を明示 | PS C:\> Get-OrchBucket -Path Orch1: -Recurse |
`

**🚨 避けるべきパターン:**
- ❌ PS Orch1:\> Get-OrchPersonalWorkspace -Path Orch1: (冗長)
- ❌ PS C:\> Get-OrchPersonalWorkspace (結果を取得できない)

**🎯 設計原則:**
1. **パラメータの価値を明示**: -Pathは「任意場所実行」を示す
2. **実用性重視**: 実際の使用シナリオを反映
3. **一貫性**: 同類パラメータは同一プロンプトパターン

### **判定方法**
`powershell
# 1. ルートフォルダに移動

# 2. パラメータなしでテスト

# 3. 結果で判定

# ❌ "Use Set-Location cmdlet" エラー → フォルダエンティティ
`
