# Get-OrchFolderMachineAccountMapping

## SYNOPSIS
Gets machine account mappings for folder-scoped machines.

## SYNTAX

```
Get-OrchFolderMachineAccountMapping [[-Name] <String[]>] [-Path <String[]>] [-Recurse] [-Depth <UInt32>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchFolderMachineAccountMapping cmdlet retrieves the account mappings between machines and user accounts within specific folders. These mappings define which users can connect to which machines in the folder context.

This is a folder entity cmdlet. To use this cmdlet, you must first navigate to the target folder using Set-Location (cd), or specify the target folders using -Path, -Recurse, or -Depth parameters.

The cmdlet returns information about machine-user relationships, including robot accounts and their associated machines within the folder scope.

Primary Endpoint: GET /odata/Folders/UiPath.Server.Configuration.OData.GetMachineRobots(folderId={folderId},machineId={machineId}), GET /odata/Robots/UiPath.Server.Configuration.OData.GetFolderRobots(folderId={folderId},machineId={machineId})

OAuth required scopes: OR.Robots or OR.Robots.Read

Required permissions: (SubFolders.View or Units.View or Jobs.Create)

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchFolderMachineAccountMapping
```

Gets all machine account mappings in the current folder.

### Example 2
```powershell
PS Orch1:\> Get-OrchFolderMachineAccountMapping Machine01
```

Gets account mappings for the machine named "Machine01" in the current folder.

### Example 3
```powershell
PS Orch1:\> Get-OrchFolderMachineAccountMapping *Prod*
```

Gets account mappings for all machines whose names contain "Prod".

### Example 4
```powershell
PS Orch1:\> Get-OrchFolderMachineAccountMapping -Recurse
```

Gets all machine account mappings from the current folder and all its subfolders.

### Example 5
```powershell
PS C:\> Get-OrchFolderMachineAccountMapping -Path Orch1:\Development,Orch1:\Testing
```

Gets machine account mappings from both Development and Testing folders.

### Example 6
```powershell
PS Orch1:\> Get-OrchFolderMachineAccountMapping | Where-Object {$_.RobotUserName -like "*service*"}
```

Gets all machine account mappings where the robot user name contains "service".

## NOTES

This cmdlet retrieves machine account mappings that define which users can connect to specific machines within folder contexts. The mapping information includes robot accounts and their associated machines, which is essential for security and access control.

Machine account mappings are folder-scoped, meaning the same machine may have different account mappings in different folders. This cmdlet requires appropriate folder permissions to access account mapping information.



Primary Endpoint: GET /odata/Folders/UiPath.Server.Configuration.OData.GetMachinesForFolder
OAuth required scopes: OR.Machines or OR.Machines.Read
Required permissions: Machines.View
