# Open-OrchLogLocation

## SYNOPSIS
Opens the folder where the HTTP logs of UiPathOrch cmdlets are output.

## SYNTAX

```
Open-OrchLogLocation [[-Path] <String>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Open-OrchLogLocation` cmdlet opens the folder containing HTTP request/response logs generated by UiPathOrch module cmdlets. These logs are useful for debugging and troubleshooting API communication with UiPath Orchestrator.

By default, the cmdlet opens the system's default log location. You can optionally specify a custom path using the `-Path` parameter to open a different location.

The log files contain detailed information about HTTP requests and responses exchanged between the UiPathOrch PowerShell module and the UiPath Orchestrator web API endpoints.

Primary Endpoint: Local utility function (no API endpoint)

OAuth required scopes: Not applicable (local utility function)

Required permissions: Not applicable (local utility function)

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Open-OrchLogLocation
```

Opens the default log folder location where UiPathOrch HTTP logs are stored.

### Example 2
```powershell
PS C:\> Open-OrchLogLocation -Path "C:\CustomLogs"
```

Opens the specified custom folder path instead of the default log location.
