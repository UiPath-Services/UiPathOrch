# Get-PmExternalApiResource

## SYNOPSIS
Gets external API resources from Platform Management.

## SYNTAX

```
Get-PmExternalApiResource [[-Name] <String[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Get-PmExternalApiResource` cmdlet retrieves external API resource information from UiPath Platform Management. External API resources define the available APIs and their associated OAuth scopes that can be accessed by external applications for integration with UiPath services.

This cmdlet returns comprehensive information about API resources including their names, display names, descriptions, and detailed scope definitions. Each API resource contains multiple scopes that define specific permissions for read and write access to different functional areas.

The scope information is essential for configuring OAuth applications that need to integrate with UiPath services, as it defines the exact permissions required for accessing specific API endpoints and functionality.

This is a Platform Management API cmdlet that works on any UiPathOrch drive (Orch1:, Orch1Tm:, Orch1Du:).

Multiple values for the -Name and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: [PLACEHOLDER - Platform Management external API resources endpoint]

OAuth required scopes: PM.ExternalApiResource or PM.ExternalApiResource.Read

Required permissions: [PLACEHOLDER - External API resources view permissions]

## EXAMPLES

### Example 1
```powershell
PS C:\> Set-Location Orch1:\
PS Orch1:\> Get-PmExternalApiResource
```

Gets all external API resources available in Platform Management.

### Example 2
```powershell
PS Orch1:\> Get-PmExternalApiResource *Orchestrator*
```

Gets API resources whose names contain "Orchestrator".

### Example 3
```powershell
PS Orch1:\> $orchApi = Get-PmExternalApiResource UiPath.Orchestrator
PS Orch1:\> $orchApi.scopes | Select-Object name, description | Format-Table
```

Gets the Orchestrator API resource and displays all its available OAuth scopes in a formatted table.

### Example 4
```powershell
PS Orch1:\> Get-PmExternalApiResource | Where-Object { $_.scopes.Count -gt 10 }
```

Gets API resources that have more than 10 scopes available.

### Example 5
```powershell
PS Orch1:\> $orchApi = Get-PmExternalApiResource UiPath.Orchestrator
PS Orch1:\> $orchApi | ConvertTo-Json -Depth 3
```

Gets the Orchestrator API resource and displays its complete structure in JSON format to explore all scopes and properties.

## NOTES
- This cmdlet uses Platform Management API and works on any UiPathOrch drive
- API resources include major UiPath services: Orchestrator, Test Manager, Document Understanding, AI Center, Process Mining, Data Service, and others
- The scopes property contains detailed OAuth scope information essential for API integration
- Common Orchestrator scopes follow the pattern OR.{Area} and OR.{Area}.Read/Write (e.g., OR.Assets, OR.Assets.Read, OR.Assets.Write)
- Use ConvertTo-Json to explore the complete scope structure and understand available permissions
- The information is crucial for configuring external applications that integrate with UiPath services via OAuth
