# Get-PmAuthenticationSetting

## SYNOPSIS
Gets the authentication settings for the organization.

## SYNTAX

```
Get-PmAuthenticationSetting [[-Path] <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The `Get-PmAuthenticationSetting` cmdlet retrieves the external identity provider and directory connection settings configured for the organization.
This includes OAuth/OIDC providers (such as Azure AD), SAML2 providers, and directory services.

The cmdlet returns a `PmAuthenticationRoot` object that contains the authentication scheme, client credentials, and detailed settings.
The `settingsExpanded` property automatically parses the JSON settings into a PowerShell Hashtable for easier access.

Primary Endpoint: GET /api/AuthenticationSetting/getAll/{partitionGlobalId}

## EXAMPLES

### Example 1: Get authentication settings
```powershell
PS Orch1:\> Get-PmAuthenticationSetting
```

This command retrieves the authentication settings for the current organization.

### Example 2: Access expanded settings
```powershell
PS Orch1:\> $auth = Get-PmAuthenticationSetting
PS Orch1:\> $auth.externalIdentityProviderDto.displayName
aad

PS Orch1:\> $auth.settingsExpanded["ServiceCertificateUsage"]
SigningAndEncryption
```

This example shows how to access the identity provider name and the expanded settings using the Hashtable.

### Example 3: Check if authentication is active
```powershell
PS Orch1:\> $auth = Get-PmAuthenticationSetting
PS Orch1:\> if ($auth.externalIdentityProviderDto.isActive) {
>>     "Authentication is active: $($auth.externalIdentityProviderDto.displayName)"
>> }
Authentication is active: aad
```

This example checks if external authentication is currently active.

## NOTES
- The `settingsExpanded` property is automatically populated by parsing the JSON from `externalIdentityProviderDto.settings`
- If JSON parsing fails, `settingsExpanded` will be an empty Hashtable
- This cmdlet is organization-scoped and returns a single authentication configuration
