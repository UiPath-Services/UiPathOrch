# Get-OrchFolderUsage

## SYNOPSIS
Gets folder usage statistics showing entity counts in UiPath Orchestrator folders.

## SYNTAX

```
Get-OrchFolderUsage [[-Path] <String[]>] [-Recurse] [-Depth <UInt32>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchFolderUsage cmdlet retrieves comprehensive usage statistics for UiPath Orchestrator folders, showing counts of various entities such as processes, robots, queues, assets, and other resources contained within each folder.

This cmdlet provides valuable insights for resource management, capacity planning, and organizational analysis by displaying how automation resources are distributed across the folder hierarchy. It can help identify heavily utilized folders, unused resources, and optimize resource allocation.

The cmdlet operates on folder entities and supports recursive traversal to analyze entire folder hierarchies. It returns detailed statistics including entity types, counts, and folder paths, making it useful for auditing and reporting purposes.

Primary Endpoint: GET /odata/Folders/UiPath.Server.Configuration.OData.GetEntityUsage(folderId={folderId})

OAuth required scopes: OR.Folders or OR.Folders.Read

Required permissions: Folders.View

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchFolderUsage
```

Gets usage statistics for the current folder.

### Example 2
```powershell
PS Orch1:\> Get-OrchFolderUsage -Recurse
```

Gets usage statistics for all folders recursively starting from the current location.

### Example 3
```powershell
PS C:\> Get-OrchFolderUsage -Path Orch1:\Production -Recurse
```

Gets usage statistics for all folders under the Production folder recursively.

### Example 4
```powershell
PS Orch1:\> Get-OrchFolderUsage -Recurse -Depth 2
```

Gets usage statistics for folders up to 2 levels deep from the current location.

## NOTES

This cmdlet operates on folder entities, which means you may need to navigate to specific folders or use the -Path parameter to target the desired location.

When using the -Recurse parameter with large folder hierarchies, the operation may take considerable time to complete. Consider using -Depth to limit the traversal scope if needed.

The usage statistics help identify resource distribution patterns and can be valuable for:
- Capacity planning and resource optimization
- Identifying unused or underutilized folders
- Monitoring resource growth and usage trends
- Organizational analysis and reporting

For performance analysis and reporting, consider exporting results to CSV format or using Select-Object to focus on specific metrics.
