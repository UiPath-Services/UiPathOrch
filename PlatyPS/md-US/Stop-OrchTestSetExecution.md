# Stop-OrchTestSetExecution

## SYNOPSIS
Stops running or pending test set executions in UiPath Orchestrator.

## SYNTAX

```
Stop-OrchTestSetExecution [-Id] <Int64[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Stop-OrchTestSetExecution cmdlet stops running or pending test set executions in UiPath Orchestrator. This cmdlet allows you to cancel test set executions that are currently running or queued for execution, which is useful for managing test automation resources and stopping tests that are no longer needed.

This is a folder entity operation. You must either navigate to a specific folder using Set-Location (cd) or use the -Path parameter to specify target folders. The -Path parameter specifies target folder paths. If not specified, the current folder will be targeted.

## EXAMPLES

### Example 1: Stop multiple test set executions
```powershell
PS Orch1:\Shared> Stop-OrchTestSetExecution 274957, 274958, 274959 -WhatIf
```

Shows what would happen if the stop operation were executed without actually stopping the test set execution.

### Example 2: Stop with confirmation prompt
```powershell
PS C:\> Stop-OrchTestSetExecution -Path Orch1:\Shared 274957 -Confirm
```

Stops the test set execution with an additional confirmation prompt before proceeding.

## NOTES

This cmdlet operates on folder entities, which means you may need to navigate to specific folders or use the -Path parameter to target test executions in the desired location.

Stopping test set executions will immediately terminate all running test cases within those test sets. This action cannot be undone, so use caution when stopping executions.

Before stopping test executions, consider using Get-OrchTestSetExecution to identify which executions are currently running and their status.

Use the -WhatIf parameter to preview the operation, especially when using patterns that might match multiple test executions.

When stopping multiple test executions, the operation is performed sequentially. Large numbers of executions may take time to process.

For Test Manager operations, ensure you are connected to the appropriate Test Manager drive (Orch1Tm:, Orch2Tm:, etc.) or use the -Path parameter to specify the correct drive.
