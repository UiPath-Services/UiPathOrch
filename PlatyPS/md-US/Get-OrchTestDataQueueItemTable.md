# Get-OrchTestDataQueueItemTable

## SYNOPSIS
Gets test data queue items in expanded table format from UiPath Orchestrator.

## SYNTAX

```
Get-OrchTestDataQueueItemTable [[-Path] <String>] [-Recurse] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchTestDataQueueItemTable` cmdlet retrieves test data queue items from UiPath Orchestrator and displays them in an expanded table format. This cmdlet is implemented as a function that transforms the internal JSON format of test data into a readable table structure.

While `Get-OrchTestDataQueueItem` returns test data in internal JSON format that cannot be displayed as a table, this cmdlet specifically addresses that limitation by expanding variable-format test data into a structured table presentation. This makes it ideal for viewing, analyzing, and reporting on test data content.

This is a folder entity operation cmdlet that requires either navigating to a specific folder containing test data queues or using the -Path or -Recurse parameters to specify target folders. The cmdlet dynamically adapts to the schema of the test data, displaying all columns and values in a user-friendly table format.

The cmdlet is particularly useful for data-driven testing scenarios where test data has varying column structures, as it can handle and display any test data schema by expanding the JSON content into appropriate table columns.

When specifying the -Path and -Recurse parameters, place them immediately after the cmdlet name. This placement ensures that autocomplete for subsequent parameters functions correctly.

Primary Endpoint: Function implementation (uses Get-OrchTestDataQueueItem internally which calls GET /odata/TestDataQueueItems)

OAuth required scopes: OR.TestData.Read

Required permissions: Folders.View, TestData.View (for test data queue item access)

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Get-OrchTestDataQueueItemTable
```

Gets test data queue items from the current folder and displays them in expanded table format.

### Example 2
```powershell
PS Orch1:\Shared> Get-OrchTestDataQueueItemTable TestDataQueue
```

Gets test data items from the specified queue named "TestDataQueue" in table format using positional parameter.

### Example 3
```powershell
PS Orch1:\Shared> Get-OrchTestDataQueueItemTable *Test*
```

Gets test data items from queues whose names contain "Test" and displays them in table format using wildcard matching.

### Example 4
```powershell
PS Orch1:\> Get-OrchTestDataQueueItemTable -Recurse
```

Gets test data queue items from the current folder and all subfolders recursively, displaying them in expanded table format.

### Example 5
```powershell
PS C:\> Get-OrchTestDataQueueItemTable -Path Orch1:\Production
```

Gets test data items from the Production folder in table format, demonstrating execution from any location.

### Example 6
```powershell
PS Orch1:\Shared> Get-OrchTestDataQueueItemTable | ConvertTo-Json -Depth 2
```

Gets test data in table format and displays the complete structure including table metadata in JSON format for detailed analysis.

## NOTES
- This is a folder entity operation cmdlet that requires navigation to a test data folder or use of -Path/-Recurse parameters
- This cmdlet is implemented as a function that transforms internal JSON test data into table format
- Unlike Get-OrchTestDataQueueItem which returns JSON format, this cmdlet provides user-friendly table output
- The table format dynamically adapts to the test data schema, displaying all available columns
- The cmdlet handles variable-format test data by expanding JSON content into appropriate table columns
- Use ConvertTo-Json to explore the complete structure including table formatting metadata
- The IsConsumed property indicates whether test data items have been used in test execution
- This cmdlet is essential for data-driven testing scenarios where readable test data presentation is needed
