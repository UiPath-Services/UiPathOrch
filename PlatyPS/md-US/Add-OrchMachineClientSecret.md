# Add-OrchMachineClientSecret

## SYNOPSIS
Adds client secrets to machines in the Orchestrator.

## SYNTAX

```
Add-OrchMachineClientSecret [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Add-OrchMachineClientSecret cmdlet adds client secrets to machines in the Orchestrator environment. Client secrets are authentication credentials that allow unattended robots to securely authenticate with the Orchestrator service.

Client secrets provide a secure authentication mechanism for unattended automation scenarios where robots need to connect to Orchestrator without user intervention. Each machine can have multiple client secrets for security rotation and backup purposes.

The cmdlet generates a new client secret for the specified machine(s) and returns the secret information that needs to be configured on the robot machines. This operation is essential for setting up secure unattended robot authentication.

Primary Endpoint: GET /odata/Machines, POST /api/clientsecrets/{machineId}

OAuth required scopes: OR.Machines

Required permissions: [PLACEHOLDER - requires verification of machine client secret management permissions]

## EXAMPLES

### Example 1
```powershell
PS Orch1:\> Add-OrchMachineClientSecret -Path Orch1:\Shared Machine01, Machine02
```

Adds new client secrets to multiple machines specified in a comma-separated list.

### Example 2
```powershell
PS C:\> Add-OrchMachineClientSecret -Path Orch1: Prod* -WhatIf
```

Shows what would happen if client secrets were added to all machines whose names start with Prod, without actually performing the operation.

### Example 3
```powershell
PS C:\> Add-OrchMachineClientSecret Machine01 -Path Orch1:, Orch2:
```

Adds a client secret to the machine named Machine01 in multiple Orchestrator environments.

### Example 4
```powershell
PS C:\> $secret = Add-OrchMachineClientSecret AutomationServer -Confirm
PS C:\> $secret | Select-Object MachineKey, ClientSecret
```

Adds a client secret with confirmation prompt and captures the returned secret information for configuration.

### Example 5
```powershell
PS C:\> Get-OrchMachine | Where-Object {$_.Type Standard} | Add-OrchMachineClientSecret -WhatIf
```

Identifies standard machines and shows what client secrets would be added without actually performing the operation.

## NOTES

This cmdlet operates on tenant entities, which means it affects machines across the entire Orchestrator tenant rather than specific folders.

The generated client secrets must be securely stored and configured on the corresponding robot machines. The secret information is sensitive and should be handled according to your organization's security policies.

Client secrets enable unattended robot authentication. Ensure that only authorized personnel have access to the generated secret information.

For security reasons, consider implementing regular client secret rotation using this cmdlet combined with Remove-OrchMachineClientSecret.

Use the -WhatIf parameter to preview the operation, especially when using wildcard patterns that might match multiple machines.

The returned MachineSecretKey object contains all necessary information for configuring robot authentication. Store this information securely immediately after generation.
