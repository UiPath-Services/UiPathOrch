# Get-OrchConnectionString

## SYNOPSIS
Gets connection string.

## SYNTAX

```
Get-OrchConnectionString [[-Path] <String[]>] [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchConnectionString cmdlet retrieves the connection string used by the Orchestrator service to connect to its database. This information is useful for system administrators who need to verify database connectivity settings, perform troubleshooting, or understand the current Orchestrator configuration.

Connection strings contain critical information about the database server, database name, authentication method, and other connection parameters. This cmdlet provides read-only access to this configuration data.

The cmdlet returns a UiPath.PowerShell.Entities.ODataValueOfString object that wraps the actual connection string value. Use the Value property or Select-Object -ExpandProperty Value to access the raw connection string.

This is typically used for diagnostic purposes and system administration tasks. The connection string information can help in troubleshooting connectivity issues or verifying that Orchestrator is configured to use the correct database.

## EXAMPLES

### Example 1
```powershell
PS C:\> Get-OrchConnectionString
```

Gets the connection string for the current Orchestrator instance. Returns a UiPath.PowerShell.Entities.ODataValueOfString object containing the database connection information.

### Example 2
```powershell
PS C:\> Get-OrchConnectionString -Path Orch1:, Orch2:
```

Retrieves connection strings from multiple Orchestrator instances. This is useful when managing multiple environments and you need to verify the database configuration for each.

### Example 3
```powershell
PS C:\> Get-OrchConnectionString | Select-Object -ExpandProperty Value
```

Gets the connection string and displays only the actual connection string value, excluding the wrapper object properties. This provides direct access to the raw connection string text.

### Example 4
```powershell
PS C:\> $result = Get-OrchConnectionString
PS C:\> $result.Value
```

Stores the connection string object in a variable and then accesses the actual connection string value. This approach is useful when you need to further process or analyze the connection string.

## NOTES

This cmdlet provides read-only access to the Orchestrator database connection string. The connection string information is sensitive and should be handled securely according to your organization's security policies.

The returned value is wrapped in a UiPath.PowerShell.Entities.ODataValueOfString object. Use the Value property or Select-Object -ExpandProperty Value to extract just the connection string text.

Connection strings typically contain sensitive information such as database server locations, database names, and authentication details. Ensure appropriate access controls are in place when using this cmdlet.

This cmdlet is primarily intended for system administrators and troubleshooting purposes. Regular users typically do not need access to connection string information.

The cmdlet operates on tenant-level settings and requires appropriate Settings.View permissions to access configuration data.



Primary Endpoint: GET /odata/Settings/UiPath.Server.Configuration.OData.GetConnectionString
OAuth required scopes: OR.Settings or OR.Settings.Read
Required permissions: Settings.View
