# Get-OrchPersonalWorkspace

## SYNOPSIS
Gets personal workspaces from UiPath Orchestrator.

## SYNTAX

```
Get-OrchPersonalWorkspace [[-Name] <String[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Get-OrchPersonalWorkspace` cmdlet retrieves personal workspace information from UiPath Orchestrator. Personal workspaces are individual automation environments assigned to specific users, providing isolated spaces for development, testing, and personal automation activities.

Personal workspaces enable individual productivity by giving users dedicated environments where they can develop, test, and manage automation projects independently without affecting shared organizational resources. Each workspace is associated with a specific user and provides isolation for development activities.

This cmdlet operates on tenant entities and provides information about workspace ownership, activity status, last login times, and exploring users. Personal workspaces support various automation development scenarios including process testing, personal automation, and isolated development environments.

Multiple values for the -Name and -Path parameters can be specified using comma-separated text that includes wildcards. Additionally, you can use autocomplete for these values by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: GET /odata/PersonalWorkspaces

OAuth required scopes: OR.Folders or OR.Folders.Read

Required permissions: Folders.View

## EXAMPLES

### Example 1
```powershell
PS Orch1:\> Get-OrchPersonalWorkspace
```

Gets all personal workspaces in the Orch1 tenant.

### Example 2
```powershell
PS Orch1:\> Get-OrchPersonalWorkspace *workspace*
```

Gets personal workspaces whose names contain "workspace" using wildcard pattern matching and positional parameter.

### Example 3
```powershell
PS C:\> Get-OrchPersonalWorkspace -Path Orch1:, Orch2:
```

Gets personal workspaces from both Orch1 and Orch2 tenants for cross-tenant analysis. Demonstrates that -Path parameter allows execution from any location.

### Example 4
```powershell
PS Orch1:\> Get-OrchPersonalWorkspace | ConvertTo-Json -Depth 2
```

Gets all personal workspaces and displays their structure in JSON format for detailed analysis of workspace properties including ownership and activity status.

## NOTES

This cmdlet operates on tenant entities and can be run from any location within an Orchestrator environment.

**Personal Workspace Information:**
- Id: Unique identifier for the workspace
- Key: Workspace key for API operations
- Name: Display name of the personal workspace
- IsActive: Whether the workspace is currently active
- OwnerId: Unique identifier of the workspace owner
- OwnerName: Display name of the workspace owner
- LastLogin: Timestamp of the last workspace access
- ExploringUserIds: Array of user IDs with exploring access

**Workspace States:**
- Active: Workspace is currently available for use
- Inactive: Workspace is disabled or suspended
- Exploring: Workspace has additional users with exploring access

**Common Use Cases:**
- Monitor personal workspace usage across the organization
- Identify inactive workspaces for cleanup or reactivation
- Track workspace ownership and access patterns
- Audit personal workspace allocation and utilization
- Generate reports on workspace activity and engagement

**Management Scenarios:**
- Capacity planning for personal workspace resources
- User onboarding and workspace provisioning tracking
- Identifying workspaces for archival or removal
- Monitoring collaborative workspace usage (exploring users)

**Important Note about Path Selection:**
When using Select-Object with the results, always include Path as the first property to identify which tenant each workspace belongs to. This is essential for managing workspaces across multiple tenants.

Use ConvertTo-Json to explore the complete workspace object structure including detailed ownership and access information.
