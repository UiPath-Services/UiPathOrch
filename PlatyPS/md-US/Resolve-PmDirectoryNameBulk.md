# Resolve-PmDirectoryNameBulk

## SYNOPSIS
Resolves directory entity names to their detailed information in UiPath Platform Management.

## SYNTAX

```
Resolve-PmDirectoryNameBulk [-EntityType] <String> [-Name] <String[]> [-Path <String[]>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Resolve-PmDirectoryNameBulk cmdlet resolves directory entity names (users, groups, or applications) to their detailed directory information in UiPath Platform Management. This cmdlet is useful for bulk resolution of multiple entity names to retrieve comprehensive directory details including identifiers, display names, organizational information, and source details.

This cmdlet accesses Platform Management APIs and works across all UiPath Orchestrator drives (Orch1:, Orch1Tm:, Orch1Du:).

Primary Endpoint: POST /api/Directory/BulkResolveByName/{partitionGlobalId}

## EXAMPLES

### Example 1: Resolve multiple users by name
```powershell
PS Orch1:\> Resolve-PmDirectoryNameBulk User user1@example.com, user2@example.com
```

Resolves multiple user names in a single operation to retrieve their directory information.

### Example 2: Resolve multiple groups
```powershell
PS Orch1:\> Resolve-PmDirectoryNameBulk Group Administrators, Users, Everyone
```

Resolves multiple group names to retrieve their directory information.

### Example 3: Resolve and examine structure
```powershell
PS Orch1:\> Resolve-PmDirectoryNameBulk User user@example.com | ConvertTo-Json -Depth 5
```

Resolves a user name and displays the complete object structure in JSON format for detailed analysis.

### Example 4: Resolve across multiple drives
```powershell
PS C:\> Resolve-PmDirectoryNameBulk -Path Orch1:, Orch2: User user@example.com
```

Resolves user names across multiple specified Orchestrator drives.

## NOTES
- This cmdlet is designed for bulk resolution operations to improve performance when resolving multiple entities
- The source property indicates the directory source (e.g., "local", "aad" for Azure Active Directory)
- Entity types must be specified exactly as "User", "Group", or "Application"
- Names must match exactly with directory entity names for successful resolution
- Platform Management cmdlets (starting with "Pm") provide cross-organizational directory information
- Use this cmdlet when you need to convert entity names to their full directory representations with identifiers
