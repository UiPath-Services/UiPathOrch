# Update-OrchCurrentUserURPassword

## SYNOPSIS
Updates the password for the current user in UiPath Orchestrator.

## SYNTAX

```
Update-OrchCurrentUserURPassword [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRIPTION
The Update-OrchCurrentUserURPassword cmdlet updates the password for the current user in UiPath Orchestrator. When executed without parameters, the cmdlet prompts the user to enter a new password and confirmation interactively. This cmdlet is useful for users who need to change their Orchestrator authentication password.

The cmdlet includes built-in confirmation prompts to ensure the new password is entered correctly.

This is a user-specific operation that affects only the currently authenticated user's credentials.

## EXAMPLES

### Example 1: Update current user password interactively
```powershell
PS C:\> Update-OrchCurrentUserURPassword
```

Prompts the user to enter a new password and confirmation, then updates the current user's password in Orchestrator.

### Example 2: Preview password update operation
```powershell
PS C:\> Update-OrchCurrentUserURPassword -WhatIf
```

Shows what would happen if the password update operation were to run, without actually executing the update.

### Example 3: Update password with confirmation prompt
```powershell
PS C:\> Update-OrchCurrentUserURPassword -Confirm
```

Prompts for both the new password and an additional confirmation before proceeding with the password update.

### Example 4: Update password across multiple drives
```powershell
PS C:\> Update-OrchCurrentUserURPassword -Path Orch1:, Orch2:
```

Updates the current user's password across multiple specified Orchestrator drives.

## NOTES
- This cmdlet prompts interactively for password input when executed without parameters
- The password prompts include:
  - Password: (for the new password)
  - Confirmation: (to confirm the new password)
- Both password entries are masked with asterisks (*) for security
- The cmdlet requires mandatory password and confirmation parameters when executed
- Use -WhatIf to preview the operation without making changes
- This operation affects only the currently authenticated user's password
- Ensure you have appropriate network connectivity to the Orchestrator instance when changing passwords
