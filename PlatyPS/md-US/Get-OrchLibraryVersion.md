# Get-OrchLibraryVersion

## SYNOPSIS
Gets specific versions of library packages from Orchestrator or external host feeds.

## SYNTAX

```
Get-OrchLibraryVersion [[-Id] <String[]>] [[-Version] <String[]>] [-HostFeed] [-Path <String[]>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
The Get-OrchLibraryVersion cmdlet retrieves specific versions of library packages from the Orchestrator environment or external host feeds. Unlike Get-OrchLibrary which returns the latest version of libraries, this cmdlet allows you to query all available versions of a specific library or retrieve a particular version.

This cmdlet is particularly useful for version management, dependency analysis, and when you need to work with specific versions of libraries for compatibility purposes. It can help identify all available versions of a library, check version history, and retrieve specific versions for deployment or rollback scenarios.

When used with the -HostFeed parameter, it can query external package repositories to find available versions before importing them into Orchestrator.

Primary Endpoint: /odata/Libraries/UiPath.Server.Configuration.OData.GetVersions(packageId='libraryId}')

OAuth required scopes: OR.Execution or OR.Execution.Read

Required permissions: Libraries.View

## EXAMPLES

### Example 1
```powershell
PS C:\> Get-OrchLibraryVersion UiPath.Excel.Activities
```

Retrieves all available versions of the "UiPath.Excel.Activities" library.

### Example 2
```powershell
PS C:\> Get-OrchLibraryVersion UiPath.Excel.Activities 2.11.4
```

Retrieves the specific version "2.11.4" of the "UiPath.Excel.Activities" library.

### Example 3
```powershell
PS C:\> Get-OrchLibraryVersion MyLibrary 1.0.*
```

Retrieves all versions of "MyLibrary" that match the pattern "1.0.*" (all 1.0.x versions).

### Example 4
```powershell
PS C:\> Get-OrchLibraryVersion -HostFeed UiPath.System.Activities
```

Retrieves all available versions of "UiPath.System.Activities" from external host feeds.

### Example 5
```powershell
PS C:\> Get-OrchLibraryVersion -Path Orch1:, Orch2: MyCustomLibrary
```

Retrieves all versions of "MyCustomLibrary" from multiple Orchestrator environments.

### Example 6
```powershell
PS C:\> Get-OrchLibraryVersion MyLibrary | Sort-Object Version -Descending | Select-Object -First 5
```

Gets all versions of "MyLibrary" and displays the 5 most recent versions.

### Example 7
```powershell
PS C:\> Get-OrchLibraryVersion MyLibrary | Where-Object {$_.IsPrerelease -eq $false} | ConvertTo-Json -Depth 3
```

Retrieves only stable (non-prerelease) versions of "MyLibrary" and displays the detailed structure in JSON format.

## NOTES

This cmdlet operates on tenant entities, which means it retrieves information from the entire Orchestrator tenant rather than specific folders.

When using the -HostFeed parameter, be aware that this retrieves information from external sources and may take longer to complete depending on network conditions and the size of the external feed.

This cmdlet is particularly useful for version management and dependency analysis. Use it to:
- Check all available versions before upgrading or downgrading libraries
- Identify prerelease versions for testing purposes
- Analyze version history and release patterns
- Verify specific versions are available before deployment

The returned objects include IsPrerelease property to distinguish between stable and prerelease versions.

Use ConvertTo-Json with appropriate depth settings to explore the complete structure of library version objects, as they contain detailed metadata that may not be visible in the default display format.
