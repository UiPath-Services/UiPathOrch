# Edit-OrchProcess

## SYNOPSIS
Opens process details for editing in the default web browser.

## SYNTAX

```
Edit-OrchProcess [[-Name] <String[]>] [-Path <String[]>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

## DESCRIPTION
The `Edit-OrchProcess` cmdlet opens the process details page in the default web browser for editing the specified process(es). This provides a convenient way to navigate from PowerShell to the Orchestrator web interface to modify process settings, view process details, or access editing features not available through the command line.

This cmdlet is useful for quickly navigating from PowerShell to the web interface to edit process configurations, update process settings, or access the visual process designer.

Multiple process names can be specified to open multiple process editing pages simultaneously. You can use autocomplete for process names by pressing [Ctrl+Space] or [Tab].

Primary Endpoint: Browser navigation to process editing page (no direct API endpoint)

OAuth required scopes: Not applicable (browser navigation)

Required permissions: Processes.Edit (for browser access to editing page)

## EXAMPLES

### Example 1
```powershell
PS C:\> cd Orch1:\MyWorkspace
PS Orch1:\MyWorkspace> Edit-OrchProcess MsgBox
```

Opens the process editing page for the "MsgBox" process in the default web browser.

### Example 2
```powershell
PS Orch1:\MyWorkspace> Edit-OrchProcess BackgroundProcess, BackgroundProcess
```

Opens process editing pages for multiple processes in separate browser tabs or windows.

### Example 3
```powershell
PS Orch1:\> Get-OrchProcess -Name *Agent* | Edit-OrchProcess
```

Gets all processes with "Agent" in their name and opens their editing pages.

### Example 4
```powershell
PS Orch1:\MyWorkspace> Edit-OrchProcess My*
```

Opens editing pages for all processes whose names start with "My" using wildcard pattern matching.

### Example 5
```powershell
PS Orch1:\> Edit-OrchProcess -Path Orch1:\Shared, Orch1:\root MsgBox
```

Opens the "MsgBox" process editing page from multiple specified folders.

## NOTES
- This is a utility cmdlet for navigating from PowerShell to the Orchestrator web interface
- The cmdlet requires an active web browser and internet connectivity to the Orchestrator instance
- Multiple process names will open multiple browser tabs or windows depending on browser settings
- The user must have appropriate permissions to edit processes in the web interface
- Wildcard patterns are supported for process names
