# Remove-OrchCredentialStore

## SYNOPSIS
Removes credential stores from the Orchestrator.

## SYNTAX

```
Remove-OrchCredentialStore [-Name] <String[]> [-Path <String[]>] [-ProgressAction <ActionPreference>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
The Remove-OrchCredentialStore cmdlet permanently removes credential stores from the Orchestrator environment. Credential stores are external systems that securely manage and provide credentials for automation processes, such as AWS Secrets Manager, Azure Key Vault, or the built-in Orchestrator database.

When a credential store is removed, the connection to the external credential management system is permanently deleted. This action cannot be undone, so careful consideration should be taken before removing credential stores, especially those that are actively used by assets or automation processes.

Before removing a credential store, ensure that no assets are currently configured to use that store, as this will break credential access for those assets.

Primary Endpoint: DELETE /odata/CredentialStores({credentialStoreId})

OAuth required scopes: OR.Settings or OR.Settings.Write

Required permissions: Settings.Delete

## EXAMPLES

### Example 1
```powershell
PS Orch1:\Shared> Remove-OrchCredentialStore TestStore
```

Removes the credential store named "TestStore" from the current Orchestrator environment.

### Example 2
```powershell
PS C:\> Remove-OrchCredentialStore -Path Orch1: Store1, Store2 -WhatIf
```

Shows what would happen if the specified credential stores were removed, without actually performing the removal.

### Example 3
```powershell
PS C:\> Remove-OrchCredentialStore -Path Orch1: Test* -Confirm
```

Removes all credential stores whose names start with "Test", prompting for confirmation before each removal.

### Example 4
```powershell
PS C:\> Remove-OrchCredentialStore -Path Orch1:, Orch2: LegacyStore
```

Removes the credential store named "LegacyStore" from multiple Orchestrator environments.

### Example 5
```powershell
PS C:\> Get-OrchCredentialStore | Where-Object {$_.Type -eq "Azure Key Vault" -and $_.IsReadOnly -eq $false} | Remove-OrchCredentialStore -WhatIf
```

Identifies non-read-only Azure Key Vault credential stores and shows what would be removed without actually performing the deletion.

### Example 6
```powershell
PS C:\> Remove-OrchCredentialStore -Path Orch1: UnusedStore -Confirm | Out-Null
```

Removes a credential store with confirmation prompt and suppresses the output.

## NOTES

This cmdlet operates on tenant entities, which means it affects credential stores across the entire Orchestrator tenant rather than specific folders.

Before removing credential stores, ensure they are not being used by any assets. You can use Get-OrchAsset to check for assets that may be using specific credential stores.

The built-in "Orchestrator Database" credential store typically should not be removed as it may be required for basic credential functionality.

Removing external credential stores (AWS Secrets Manager, Azure Key Vault, etc.) will break the connection to those external systems, but does not affect the credentials stored in the external systems themselves.

Use the -WhatIf parameter to preview the operation before actual execution, especially when using wildcard patterns that might match multiple credential stores.

The removal operation is irreversible. Once a credential store is deleted, it cannot be recovered and must be recreated with all its configuration if needed.

Read-only credential stores may have special restrictions on removal depending on their configuration and usage.
